# PRD: 성혼 관리 모듈

> 버전: 1.0 | 날짜: 2026-05-19 | 담당: 법무팀 / 관리자

---

## 1. 모듈 개요

| 항목 | 내용 |
|------|------|
| **모듈명** | 성혼 관리 (MN-400) |
| **대상 사용자** | 법무팀, 관리자 (일반 매니저 접근 제한) |
| **핵심 목적** | 성혼사례금 납부 관리, 법무 진행 기록, 소송중 회원 격리 관리 |
| **관련 테이블** | TB_MARRIAGE_HISTORY, TB_CALL_LOGS, TB_LEGAL_MEMOS |

---

## 2. 대상 회원 정의

| 조건 | 설명 |
|------|------|
| 진입 조건 | 정회원 → 소송중 상태 전환 시 자동 등록 |
| 상태값 | 확인전, 확인중, 미혼, 미팅중, 위임서류 미비, 성혼비 없음, 소송중, 외부결혼, 성혼, 결혼예정, 납부완료 |
| 접근 제한 | 소송중 회원은 일반 매니저의 상세 페이지 접근 차단 |

---

## 3. 기능 상세

### 3.1 성혼 관리 목록 (MN-400)

**검색 필터:** 이름, 회원ID, 성별, 브랜드, 프로그램, 성혼확인 상태, 납부상태, 매칭매니저, 지역

**목록 컬럼:**

| 컬럼 | 설명 |
|------|------|
| 체크박스 | 일괄 선택 |
| 이름 | 회원명 (클릭 시 관리 모달) |
| ID | 회원번호 |
| 성별 | 남/여 |
| 나이 | 세 |
| 브랜드 | 퍼플스/디노블/르매리 |
| 프로그램 | 가입 프로그램명 |
| 성혼확인 | 11단계 상태값 |
| 성혼비 | 금액 |
| 납부상태 | 미납/납부중/완납 |
| 매칭매니저 | 담당자 |
| 지역 | 거주지역 |
| 가입일 | 정회원 가입일 |

**페이지네이션:** 페이지당 20건 표시

**부가 기능:**
- CSV 내보내기 (이름, ID, 성별, 나이, 브랜드, 프로그램, 성혼확인, 성혼비, 납부상태, 매칭매니저, 지역, 가입일)
- 페이지 진입 시 안내 팝업 자동 표시

### 3.2 관리 모달 — 2탭 구조

회원 이름 클릭 시 관리 모달 표시. **이력변경**과 **이력조회** 2개 탭으로 구성.

#### 탭 1: 이력변경

| 섹션 | 기능 | 설명 |
|------|------|------|
| 성혼확인 상태 변경 | 드롭다운 + 사유 입력 | 11단계 상태값 중 선택, 사유 필수 입력 후 저장 |
| 통화 기록 작성 | 유형 + 응대결과 + 내용 | 유형: 납부안내, 지급요청, 법적안내, 기타 |
| 법무 메모 작성 | 메모 텍스트 입력 | 일반 매니저 메모와 분리된 법무 전용 메모 |

**통화 기록 유형:**

| 유형 | 설명 |
|------|------|
| 납부안내 | 성혼사례금 납부 안내 통화 |
| 지급요청 | 성혼사례금 지급 요청 |
| 법적안내 | 법적 진행 사항 설명 |
| 기타 | 기타 상담 내용 |

**응대 결과:** 완료, 부재, 콜백예정, 거부

#### 탭 2: 이력조회

| 기능 | 설명 |
|------|------|
| 통합 타임라인 | 상태변경 이력 + 통화기록 + 법무메모를 날짜 역순 통합 정렬 |
| 구분 표기 | 각 이력 유형별 라벨로 구분 (상태변경/통화/메모) |
| 상세 정보 | 변경일시, 담당자, 내용, 사유 등 |

### 3.3 납부완료 보고

| 항목 | 내용 |
|------|------|
| 트리거 | 납부완료 보고 버튼 클릭 |
| 프로세스 | 모달 확인 → 회장님 대시보드 자동 상신 |
| 승인 후 | 최종 성혼 확정, 상태 Lock |

## 4. 접근 제어 정책

| 역할 | 권한 |
|------|------|
| 일반 매니저 | 소송중 회원 상세 페이지 접근 **차단** (경고 팝업 표시 후 차단) |
| 법무팀 | 전체 접근, 통화기록/메모 작성 가능 |
| 관리자 | 전체 접근, 상태 변경, 납부완료 보고 |
| 회장님 | 납부완료 승인 결재 |

**소송중 회원 접근 차단 팝업:**
- 제목: "법무팀 전담 응대 회원"
- 내용: "연락 및 결제 진행을 즉시 중단하세요."
- 접근 허용 역할: admin, legal, director

---

## 5. 진입 안내 팝업

성혼 관리 페이지 최초 진입 시 4가지 안내 항목 표시:

| No. | 항목 | 설명 |
|-----|------|------|
| 1 | 소송중 회원 접근 차단 | 소송중 상태의 회원은 일반 매니저의 상세 페이지 접근이 차단됩니다 |
| 2 | 성혼확인 상태 관리 | 확인전, 확인중, 성혼, 소송중 등 11단계 상태를 관리합니다 |
| 3 | 성혼비 납부 관리 | 프로그램별 성혼비 납부 현황 확인 및 납부완료 보고가 가능합니다 |
| 4 | 회장님 승인 플로우 | 납부완료 보고 시 회장님 대시보드로 자동 상신, 승인 시 최종 성혼 확정됩니다 |

## 7. 데이터 모델 (프로토타입)

현재 `DUMMY` 객체 기반 프로토타입 운영 중. 향후 Supabase 전환 예정.

| 테이블 | 주요 필드 |
|--------|----------|
| marriage_history | id, member_id, from_status, to_status, reason, changed_at, changed_by |
| call_logs | id, member_id, call_type, result, content, called_at, caller |
| legal_memos | id, member_id, content, created_at, author |

## 8. 향후 작업 계획

| 우선순위 | 작업 | 설명 |
|----------|------|------|
| P1 | Supabase 연동 | DUMMY → 실제 DB 전환 (marriage_history, call_logs, legal_memos) |
| P1 | 결재 API 연동 | 납부완료 보고 → 회장님 승인 결재 엔드포인트 |
| P2 | 권한 분리 | 매니저/법무팀 역할별 열람/작성 권한 제어 |
| P2 | CSV 고도화 | 통화기록 및 법무메모 데이터 포함 내보내기 |
