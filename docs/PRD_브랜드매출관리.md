# 브랜드별 매출 관리 설계서

> **작성일**: 2026-05-12  
> **상태**: 확정  
> **관련**: 매출 현황 조회, 매출 통계, 정산

---

## 1. 배경 및 현행 문제점

### 1.1 기존 시스템 구조

| 항목 | 현황 | 문제점 |
|------|------|--------|
| 시스템 | 퍼플스/디노블/르매리 각각 별도 인트라넷 운영 | 데이터 사일로, 동일 회원 이중 등록 |
| 회원 DB | 각 인트라넷에 타 브랜드 회원도 혼재 | 브랜드 구분 없이 관리 |
| 매출 집계 | 대전·대구·광주는 전부 퍼플스 매출로 합산 | 브랜드별 실제 수익성 파악 불가 |
| 데이터 정합성 | 동일 회원이 다른 정보로 각 시스템에 등록 | 연동 없음, 어느 쪽이 최신인지 불명 |

### 1.2 개선 방향

**통합 인트라넷 1개로 전 브랜드 관리**, 매니저 소속 브랜드 기준으로 매출 자동 귀속

## 3. 매출 귀속 핵심 규칙

### 3.1 원칙: 매니저 소속 브랜드 기준

> **매출은 회원의 브랜드가 아닌, 담당 매니저의 소속 브랜드로 귀속된다.**

```
매니저 테이블
┌──────────┬──────────┬──────────┐
│ 매니저명  │ 소속 지사  │ 소속 브랜드 │
├──────────┼──────────┼──────────┤
│ 이지연    │ 본사      │ 퍼플스     │
│ 박수정    │ 본사      │ 디노블     │
│ 최영미    │ 대전      │ 퍼플스     │  ← 디노블 회원 담당해도 퍼플스 매출
│ 한소영    │ 부산      │ 디노블     │
└──────────┴──────────┴──────────┘
```

### 3.2 단독 매출

```
회원: 김서연 (디노블)
매니저: 이지연 (퍼플스본사)
매출: 800만원

→ 매출 귀속: 퍼플스 800만원 (매니저 소속 기준)
```

### 3.3 쉐어 매출 (동일 브랜드)

```
회원: 박지유 (퍼플스)
매니저: 이지연 (퍼플스) 60% / 정다은 (퍼플스) 40%
매출: 1,000만원

→ 퍼플스 매출: 1,000만원
  - 이지연 실적: 600만원
  - 정다은 실적: 400만원
```

### 3.4 크로스 브랜드 쉐어

```
회원: 최수아 (디노블)
매니저: 박수정 (디노블본사) 60% / 이지연 (퍼플스본사) 40%
매출: 800만원

→ 비율대로 브랜드 분배:
  - 디노블 매출: 480만원 (박수정 60%)
  - 퍼플스 매출: 320만원 (이지연 40%)
```

## 4. 데이터 모델

### 4.1 매니저 테이블

```
Manager {
  id          : 매니저 코드 (PK)
  name        : 매니저명
  branch_id   : 소속 지사
  brand_code  : 소속 브랜드 (PUR/DNB/LMR)  ← 핵심
  is_active   : 활동 여부
}
```

### 4.2 매출 테이블

```
Sales {
  id              : 매출 ID (PK)
  member_id       : 회원 ID
  member_brand    : 회원 소속 브랜드
  pay_date        : 입금일
  contract_date   : 계약일
  amount          : 입금액
  split_type      : 분할매출 (일시불/계약금/중도금/잔금/업그레이드)
  join_order      : 가입순서 (1가입/2가입)
  male_type       : 남성수당 (전문직/재력가/프로모션/없음)
  manager_id      : 메인 매니저 코드
  manager_brand   : 메인 매니저 소속 브랜드 (자동)
  share_manager_id: 쉐어 매니저 코드
  share_brand     : 쉐어 매니저 소속 브랜드 (자동)
  share_ratio     : 쉐어 비율
  confirmed       : 확인 여부
}
```

### 4.3 매출 조회 쿼리 로직 (의사코드)

```
브랜드별 매출 집계:

FOR EACH 매출 IN 매출테이블:
  IF 쉐어 없음:
    매니저.소속브랜드 += 매출.금액
  ELSE:
    메인매니저.소속브랜드 += 매출.금액 × (100 - 쉐어비율)%
    쉐어매니저.소속브랜드 += 매출.금액 × 쉐어비율%
```

## 5. UI 반영 사항

### 5.1 매출 현황 조회 (input.js)

- 브랜드 필터: 매니저 소속 브랜드 기준
- 쉐어 표시: 크로스 브랜드 쉐어 시 매니저 옆에 브랜드 뱃지 표시

### 5.2 매출 통계 (stats.js)

- 브랜드별 매출 카드: 매니저 소속 기준 집계
- 크로스 브랜드 쉐어 건은 비율대로 분배하여 각 브랜드에 반영

### 5.3 매니저 설정

- 매니저 등록/수정 시 **소속 브랜드** 필수 선택
- 지사 변경 시 소속 브랜드 자동 확인 (대전·대구·광주 → 퍼플스만 허용)

## 6. 마이그레이션 고려사항

| 항목 | 처리 방안 |
|------|----------|
| 기존 동일 회원 이중 등록 | 주민번호 기준 중복 병합, 최신 정보 우선 |
| 대전·대구·광주 디노블/르매리 매출 | 기존대로 퍼플스 매출 유지 (변경 없음) |
| 매니저 소속 브랜드 | 기존 지사 정보 기반으로 일괄 설정 |
| 부산 브랜드 분리 데이터 | 매니저 코드 매핑으로 자동 분리 |
