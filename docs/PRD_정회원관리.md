## 3. 기능 상세

### 3.1 정회원 목록 (MN-300)

**목록 컬럼:** 회원번호, 이름, 성별, 나이, 프로그램, 상태, 브랜드, 매칭매니저, 가입일, 만료일

### 3.3 계약 관리 (MN-302)

**3대 계약 형태:**

| 계약 형태 | 계약 방법 | 생애주기 | 적용 프로그램 |
|----------|----------|---------|-------------|
| 인증제(1장) | 기간 | 기간 종료 → 만료 | 전문직(55만↓) |
| 인증제(2장) | 횟수/기간+기간 | 횟수or기간 종료 → 활동대기 | 전문직(100만↑), 준전문직 |
| 횟수제 | 횟수+기간 | 횟수 종료 → 활동대기 | 정규, 기타, 프로모션 |
| 기간제 | 기간+횟수 | 기간 종료 → 만료 | 정규, 기타, 프로모션 |

**핵심 규칙:**
1. 계약 시작일 = 전산 생성일 **(절대 수정 불가, 시스템 Lock)**
2. 만료일 = 룰 엔진 자동 계산 (프로그램별)
3. 1가입/2가입 통합 릴레이션 DB 관리

### 3.4 프로필 발송 규칙 (MN-311)

| 조건 | 시스템 처리 |
|------|-----------|
| 신규/인증중/활동대기 상태 | 발송 기능 비활성화 (URL 생성 차단) |
| 활동 상태만 | 발송 허용 (매칭 매니저만) |
| 72시간 무응답 | 자동 거절 처리 (RL-011) |
| 소송중 전환 | 프로필 URL 무효화 + 전면 차단 |
| 교제 중 | 타 회원 발송 시스템 차단 |
| 보류 중 | 발송 일시 중지 (복귀 시 재개) |
| 사진 보정 금지 | 매니저 과도한 보정 업로드 엄격 금지 |

## 4. 업그레이드/다운그레이드 정책

| 시점 | 처리 방식 | 비고 |
|------|----------|------|
| 인증 전 변경 | 기존 계약 취소 + 신규 계약서 재작성, 처음부터 재진행 | 계약서·결제·서류 전부 초기화 |
| 인증 후 변경 | 전산 내 단계 업그레이드 기능 활용, 매칭 이력 유지 | 차액 정산 자동 계산 |

## 5. 탈회 접수 차단 정책

| 차단 상태 | 사유 | 시스템 처리 |
|----------|------|-----------|
| 활동대기 | 활성 서비스 미진행 | 탈회 버튼 Disabled |
| 만료(기간만료) | 이미 서비스 종료 | 탈회 버튼 Disabled |
| 자동만료 | 시스템 자동 만료 완료 | 탈회 버튼 Disabled |
| 약정만료 | 횟수 소진 완료 | 탈회 버튼 Disabled |
| 탈회 | 이미 탈회 완료 | 재접수 불가 안내 |

> **예외**: 본부장급 이상 [관리자 강제 접수] + 사유 필수 → 감사 로그 자동 기록
