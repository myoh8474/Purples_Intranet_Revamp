# 트리니티 회원 관리 설계서

> **작성일**: 2026-05-13  
> **상태**: 확정  
> **관련**: 정회원 관리, 바우처 발송, 등급 관리

---

## 1. 배경 및 개요

### 1.1 트리니티 회원이란

- **다이아몬드 등급 이상** 정회원이 트리니티 회원 자격을 획득
- 정회원 가입 후 등급이 다이아몬드 이상일 경우 **이벤트 문자 발송** 대상이 됨
- 바우처 안내 문자 발송 시 **자동으로 트리니티 게시판에 등록**

### 1.2 현행 업무 프로세스

```
[정회원 가입] 다이아몬드 등급 이상 확인
   ↓
[정회원 상세 페이지] "이벤트문자 발송" 버튼 활성화
   ↓
[담당자 클릭] 이벤트문자 발송 버튼 클릭
   ↓
[서버] 바우처 안내 문자 발송 (SMS/LMS)
   ↓
[자동] 트리니티 회원 게시판에 해당 회원 자동 저장
   ↓
[인포/매니저] 트리니티 관리 페이지에서 후속 관리
  - 1회 미팅일 기록
  - 수령증 발송 여부
  - 상품권 발송 여부
  - 메모 기록
```

## 3. 데이터 모델

### 3.1 트리니티 회원 테이블

```
TrinityMember {
  id              : 트리니티 ID (PK, 자동채번)
  branch          : 지사 (본사/경기/부산/대전/대구/광주)
  member_id       : 정회원 ID (FK → RegularMember)
  name            : 회원명
  gender          : 성별 (남/여)
  birth_date      : 생년월일
  program         : 프로그램명 (가입 상품)
  phone           : 연락처
  consult_mgr     : 상담매니저
  match_mgr       : 매칭매니저
  join_date       : 가입일 (정회원 가입일)
  voucher_date    : 바우처 발송일 (이벤트문자 발송일)
  first_meeting   : 1회 미팅일
  receipt_sent    : 수령증 발송 여부 (Y/N)
  gift_sent       : 상품권 발송 여부 (Y/N)
  memo            : 메모
  created_at      : 등록일시 (자동)
}
```

- 현재 필터 적용된 리스트를 **XLSX 형식으로 다운로드**
- 파일명: `트리니티회원_YYYYMMDD.xlsx`
- 모든 컬럼 포함

- 메모 컬럼의 아이콘 클릭 시 **인라인 입력 또는 모달**로 메모 작성/수정

## 6. 트리니티 자동 등록 트리거

### 6.1 발동 조건

```
IF 정회원.등급 ∈ [다이아몬드, 로얄다이아, VIP, VVIP]
  AND 이벤트문자발송 버튼 클릭
THEN
  1. 서버에서 바우처 안내 SMS 발송
  2. 발송 성공 시 트리니티 테이블에 INSERT
  3. voucher_date = 발송 일시
```

### 6.2 정회원 상세 페이지 연동

- 다이아몬드 이상 회원의 상세 페이지에만 **[이벤트문자 발송]** 버튼 표시
- 이미 발송된 경우 **[발송완료]** 상태로 비활성화
