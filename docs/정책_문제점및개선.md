## 1. 현행 문제점 분석 (4개 카테고리, 23건)

### 1.1 메뉴 구조 (3건)

| 심각도 | 문제 | 설명 |
|--------|------|------|
| 🔴 Critical | 메뉴 파편화 | 동일 업무가 여러 메뉴에 분산되어 부서별 데이터 사일로 발생 |
| 🔴 Critical | 중복 기능 | 회원정보 조회가 상담팀/매칭팀/CS팀 메뉴에 각각 존재하여 데이터 불일치 |
| 🟠 High | 비직관적 네비게이션 | 업무 흐름과 메뉴 배치가 불일치하여 신규 입사자 교육 비용 증가 |

### 1.2 데이터 관리 (7건)

| 심각도 | 문제 | 설명 |
|--------|------|------|
| 🔴 Critical | 상태값 모호성 | 회원 상태(활동/보류/교체/탈퇴 등)의 정의가 불명확하고 수동 변경으로 오류 빈발 |
| 🔴 Critical | 계약일 임의 조작 | 매니저가 실적 이월 목적으로 계약 시작일을 임의 변경 가능 → 위약금/환불금 산정 오류 |
| 🔴 Critical | 블랙리스트 통제 부재 | 서류 위조/신원 부적격자 적발 후 수동 기록만 가능, 재가입 원천 차단 시스템 없음 |
| 🔴 Critical | 개인정보 파기 미수립 | 탈회/종료 후 데이터 수동 관리로 법정 보유 기간 준수 여부 확인 불가 |
| 🟠 High | 이력 추적 불가 | 상태 변경, 매칭 이력, 결제 이력의 감사 로그(Audit Log) 부재 |
| 🟠 High | 데이터 연동 부재 | 결제/인증/매칭 시스템 간 데이터가 분리되어 수동 대조 필요 |
| 🟠 High | 재컨텍 DB 출처 미구분 | 정회원 만료·가입보류·장기상담 등 재유입 DB의 출처가 구분 안 됨 |

### 1.3 업무 프로세스 (9건)

| 심각도 | 문제 | 설명 |
|--------|------|------|
| 🔴 Critical | 중복DB 수동 처리 | 동일 고객 중복 유입 시 수동 확인/소스외 처리 → 콜 겹침 및 영업 충돌 |
| 🔴 Critical | 잔금미납 방치 | 계약만 체결 후 잔금 미납되어도 무기한 방치 → 허수 계약 누적 |
| 🔴 Critical | 프로필 무분별 발송 | 미인증/보류/소송중 회원에게도 프로필 발송 가능 → 개인정보 유출 위험 |
| 🟠 High | 수동 프로세스 과다 | 정회원 전환, 서류 인증, 프로필 발송 등 대부분 수작업 처리 |
| 🟠 High | 장기상담 DB 홀딩 | 매니저가 가망 고객 DB를 무기한 보유 가능 → 영업 기회 차단 |
| 🟠 High | 탈회 처리 SLA 부재 | 탈회 접수 후 환불 정산 처리 기한 없음 → 무기한 지연 |
| 🟠 High | 분배 후 미컨텍 방치 | 분배 후 매니저가 첫 연락 미시도해도 통제 수단 부재 |
| 🟡 Medium | 결재선 부재 | 중요 변경사항에 대한 체계적 결재/승인 프로세스 없음 |
| 🟡 Medium | 알림 체계 미흡 | 자동 알림(카카오톡, SMS) 발송 체계 부재 |

### 1.4 인증/서류 (3건)

| 심각도 | 문제 | 설명 |
|--------|------|------|
| 🟠 High | 수동 서류 수집 | 회원 인증 서류를 매니저가 직접 수집/등록 → 누락 및 지연 |
| 🟠 High | API 미연동 | 공공기관 스크래핑 API 미연동 |
| 🟡 Medium | 인증 상태 불투명 | 서류별 인증 진행 상태를 한눈에 파악하기 어려움 |

## 2. 개선 방안 (21건)

| # | 영역 | Before | After | 임팩트 | 우선순위 |
|---|------|--------|-------|--------|----------|
| 1 | 계약 시작일 | 매니저가 임의로 입력/변경 가능 | 인포팀 전산 생성일 = 계약 시작일 자동 Lock | 실적 이월 방지, 위약금 정확도 100% | Critical |
| 2 | 결제 체계 | 1차/2차 가입비 분리 관리 안 됨 | 1가입/2가입 분리 + 통합 릴레이션 DB | 결제 이력 추적, 환불 자동화 | Critical |
| 3 | 중복DB 수거 | 수동으로 중복 확인 후 소스외 처리 | 휴대폰 번호 기반 실시간 중복 탐지 → 자동 이관 + 알림 | 반복 업무 100% 자동화 | Critical |
| 4 | 채널별 유입 | 수동 엑셀 다운로드 후 직접 입력 | API/웹훅 자동 연동 + 실시간 로그 + 스팸 자동 차단 | DB 유입 즉시 분배 가능 | Critical |
| 5 | 블랙리스트 | 수동 기록만 가능, 재가입 차단 부재 | UID 기반 영구 차단 + 해시 대조 → 원천 차단 | 부적격자 재유입 차단 | Critical |
| 6 | 잔금미납 | 무기한 방치 | 14일 경고 → 30일 자동 탈회진행 전환 | 허수 계약 제거 | Critical |
| 7 | 프로필 발송 통제 | 미인증/보류/소송중에도 발송 가능 | 상태별 Lock + 소송 시 URL 무효화 + 교제 시 차단 | 개인정보 유출 차단 | Critical |
| 8 | 개인정보 파기 | 수동 관리, 법정 기간 준수 불확실 | 정회원 5년/준회원 1년 → 매월 자동 파기 + 파기 로그 | 개인정보보호법 준수 | Critical |
| 9 | 회원 상태 | 수동 변경, 프로필 발송 수동 제어 | 상태값 자동 변경 + 프로필 발송 Lock 자동화 | 매칭 오류 방지 | High |
| 10 | 회원 상세 | 부서별 별도 화면에서 분산 조회 | 원페이지 통합 회원 상세 뷰 | 정보 사일로 해소 | High |
| 11 | 서류 인증 | 매니저가 직접 요청/수집 | 스크래핑 API 자동 발급 + 간편인증 | 수집 시간 90% 단축 | High |
| 12 | 알림 체계 | 수동 문자/카톡 발송 | 상태별 자동 알림톡/SMS 발송 | 누락 방지 | High |
| 13 | 장기상담 제어 | 무기한 보유(홀딩) 가능 | 90일×2회 → 만료 시 강제 회수 + 60일 쿨타임 | DB 독점 방지 | High |
| 14 | 가입보류 제어 | 무기한 유지 가능 | 30일×1회 → 미결제 시 태그 + DB 풀 방출 | 결제 전환 촉진 | High |
| 15 | 탈회 SLA | 처리 기한 없음 | 7영업일 내 정산 완료 + SLA 경고 자동 발송 | 법적 분쟁 차단 | High |
| 16 | 재컨텍 출처 관리 | 출처 미구분, 랜덤 분배 | 4종 출처 태그 + 우선순위별 분배 + 쿨타임 | 고가치 DB 우선 처리 | High |
| 17 | 분배 상한선 | 무제한 DB 배정 가능 | 일일 20건/월간 200건 + 48시간/72시간 미컨텍 경고 | 균등 배분, DB 방치 제로 | High |
| 18 | 서류 반려 | 매니저 수동 확인, 별도 게시판 | 반려 즉시 자동 알림 + 7일 보완 기한 + 미보완 시 파기 | 반려 처리 속도 향상 | High |
| 19 | CS/환불 | 클레임→탈회→환불 각각 수동 처리 | 원스톱 파이프라인 + 전자서명 내재화 | 처리 시간 단축 | Medium |
| 20 | 매칭 필터 | 수동 검색/선별 | 등급 +1단계 제한 자동 필터링 | 매칭 품질 향상 | Medium |
| 21 | 업그레이드/다운그레이드 | 절차 미수립 | 인증 전: 계약 취소 + 재작성 / 인증 후: 단계업그레이드 | 정산 정확도 향상 | Medium |
