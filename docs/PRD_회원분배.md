## 2. 페이지 구성

### 2.1 회원분배 (`distribute.html`)

**역할**: 미분배 회원을 매니저에게 배정하는 **작업(쓰기)** 페이지

| 구성 요소 | 설명 |
|----------|------|
| **3탭 구조** | 신규 회원 분배 / 중복 회원 분배 / 기간만료(재컨텍) |
| **검색 필터** | 3행 테이블: 회원검색(이름/연락처), 상세검색(날짜/경로/학력/성별), 매니저 칩 |
| **액션 버튼** | 매니저 드롭다운(보유DB 표시) + 상담매니저 등록 + 소스외 처리 |
| **리스트** | 체크박스 선택, 탭별 컬럼 구성, 페이지네이션 20건 |

**분배 흐름**:
```
체크박스로 회원 선택 → 드롭다운에서 매니저 선택 → [상담매니저 등록] 클릭 → confirm → 완료
```

### 2.2 회원분배내역조회 (`distribute-history.html`)

**역할**: 분배 완료 회원을 **조회(읽기)** 하는 페이지

| 구성 요소 | 설명 |
|----------|------|
| **검색 필터** | 5행 테이블: 회원검색, 상세검색(유입구분/경로/성별/회원상태), 등록일, 분배일, 매니저 |
| **리스트** | 11컬럼(No/이름/유입구분/성별/나이/연락처/유입경로/등록일/분배일/담당매니저/현재상태) |
| **정렬** | 등록일/분배일 컬럼 클릭 정렬 (기본: 분배일 내림차순) |
| **페이지네이션** | 20건 단위 |

| 재컨텍 DB | [기간만료(재컨텍)] + 쿨타임 완료 | 출처 태그별 분배 |

## 4. 스크리닝 규칙 (`screening.service.js`)

| 단계 | 검증 항목 | 처리 |
|------|----------|------|
| 1 | 전화번호 형식 검증 | 형식 오류 → 자동 소스외 (block) |
| 2 | 기간만료(재컨텍) 여부 | channel='기간만료(재컨텍)' → 재컨텍 탭 |
| 3 | 중복 DB 검사 (전화번호) | 기존 진행중 회원과 동일 → 중복 탭 |
| 4 | 신규 분류 | 위 해당 없음 → 신규 탭 |

> 나이/학력은 차단하지 않음 — 팀장이 데이터를 보고 직접 판단

## 5. 분배 방식

### 5.1 상담매니저 등록 (수동 분배)
- 팀장급 이상만 실행 가능
- 미분배 회원 체크 → 매니저 드롭다운 선택 → 등록 버튼
- 등록 시 담당매니저 배정 → 준회원 리스트에 `컨텍전` 상태로 등록
- `distributedAt` 현재 시간 기록
- localStorage로 페이지 간 상태 동기화

### 5.2 자동 분배

#### 5.2.1 홈페이지 지정 상담사 루트 (확정)
- 홈페이지에서 특정 상담사 프로필을 통해 유입된 DB는 **해당 상담사에게 자동 분배**
- 유입 시점에 지정된 상담사 정보(consultant_id)가 DB에 기록됨
- 스크리닝(전화번호 검증, 중복 검사) 통과 후 **즉시 자동 배정** → `컨텍전` 상태로 등록
- 지정 상담사가 퇴사/휴직 상태인 경우 → **미분배 대기열**로 이동 (수동 분배 대상)

#### 5.2.2 랜딩페이지 유입 DB — ⚠️ 자동분배 여부 확인 필요
- 랜딩페이지(마케팅 광고 등)로 유입된 DB의 자동 배정 여부 **검토 필요**
- **자동분배 적용 시**: 활성 매니저 순서대로 1건씩 균등 배정 (라운드로빈)
- **수동분배 유지 시**: 기존과 동일하게 팀장이 직접 매니저 선택 후 배정
- 동일 지사 소속 매니저에게만 분배 (지사 격리)
- 📌 협업 부서 확인 후 최종 결정 예정

## 6. 중복 DB 수거

| 단계 | 처리 |
|------|------|
| 1 | 유입 DB 휴대폰 번호 → 기존 전체 DB 대조 |
| 2-A (고유) | 정상 분배 대기열 등록 |
| 2-B (중복) | 중복 회원 탭으로 분류 + 기존 매니저 정보 표시 |
| 3 | 중복 수거 이력 자동 기록 (원본 연결 보존) |
