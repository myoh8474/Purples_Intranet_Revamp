## 2. 회원 상태 정의 (15개 상태)

### 2.1 상태값 목록

| 단계 | 상태명 | 설명 | 자동화 룰 |
|------|--------|------|----------|
| 연락전 | 컨텍전 | 신규 DB 유입, 매니저 배정 및 첫 연락 전 | - |
| 상담진행 | 부재중(미연락) | 통화 미연결 | 3회 부재 → 결번(불가) |
| 상담진행 | 낮음(컨텍) | 가입 의사 낮음 | - |
| 상담진행 | 보통(컨텍) | 일반 상담 진행 | - |
| 상담진행 | 높음(컨텍) | 가입 의사 높음 | - |
| 상담진행 | 장기상담(컨텍) | 장기적 관리 필요 | 최종컨텍일 3개월 경과 → 컨텍전 원복 |
| 방문상담 | 방문상담 | 오프라인 상담 진행 | - |
| 가입/보류 | 가입보류(컨텍) | 가입 결정 연기 | 최종컨텍일 3개월 경과 → 컨텍전 원복 |
| 가입/보류 | 가입중 | 계약/결제 진행 중 | 14일 경고/30일 파기 |
| 가입/보류 | 가입완료 | 정회원 이관 대기 | - |
| 진행불가 | 결번(불가) | 연락처 오류 | 영구 차단 |
| 진행불가 | 기혼(불가) | 법적 서비스 불가 | 영구 차단 |
| 진행불가 | 기타(불가) | 블랙리스트 등 | 영구 차단 (§7장) |
| DB정리 | 변경 | 타 지사 이관 등 | 팀장 승인 필수 |
| 만료 | 기간만료(재컨텍) | 영업 종료 후 재진입 | 출처태그+쿨타임 |

### 2.2 상태 전이 규칙

- `컨텍전` → 첫 통화 시도 → `부재중` 또는 `낮음/보통/높음`
- `부재중` → 3회 누적 → `결번(불가)` (영구)
- `높음/보통` → 미팅 예약 → `방문상담`
- `방문상담` → 가입 진행 → `가입중` → 전액 결제 → `가입완료` → **정회원 전환**
- `장기상담` / `가입보류` → 최종컨텍일 기준 **3개월 경과** → `컨텍전` 상태로 자동 원복

## 3. 기능 상세

### 3.1 준회원 목록 (list.html)

| 항목 | 내용 |
|------|------|
| **파일** | `pages/associate/list.html` |
| **페이지 ID** | `associate-list` |
| **목적** | 전체 준회원 목록 조회 및 개별 상세 접근 |

**검색 필터 (멀티셀렉트 방식):**
- 회원검색 (이름/전화번호/직장 키워드)
- 성별, 결혼여부, 학력, 상태, 가입경로, 지사, 담당매니저
- 전체 필터는 **커스텀 체크박스 드롭다운**으로 복수 조건 선택 가능
- 전체 초기화 버튼 제공

**테이블 컬럼 (16개):**
| 번호 | 지사 | 등록일 | 분배일 | 최종컨텍일 | 회원명 | 생년 | 성별 | 결혼여부 | 가입경로 | 학력 | 연락처 | 직업 | 지역 | 회원상태 | 담당자 |

**기능:**
1. 회원명 클릭 → 준회원 상세(detail.html) 새 탭 열기
2. 멀티셀렉트 필터링 (복수 조건 AND 적용)
3. 체크박스 선택 → 상단 선택건수 표시 바
4. 페이지당 20건 + 페이지네이션
5. 본인 담당 회원만 조회(MGR) / 소속팀 전체(TL) / 전체(ADMIN)
6. Excel 다운로드 (TL 이상)
7. 지사 옵션은 공통 상수(`BRANCHES`)로 관리

### 3.2 상담통계 (consult-status.html)

| 항목 | 내용 |
|------|------|
| **파일** | `pages/associate/consult-status.html` |
| **페이지 ID** | `associate-consult-status` |
| **목적** | 기간별 매니저·지사 상담 활동 통계 리포트 |

**1. 검색 필터:**
- 1_1) 기간 검색: 시작일 ~ 종료일 (date picker)
- 1_2) 퀵버튼: 오늘 / 이번주 / 이번달 클릭 시 기간 자동 설정
- 1_3) 검색 버튼 클릭 시 KPI·지사별·상담사별 데이터 전체 갱신
- 1_4) 초기화 버튼 클릭 시 기간을 오늘로 리셋

**2. 엑셀 다운로드:**
- 2_1) KPI 카드 상단 우측에 다운로드 버튼 배치
- 2_2) 클릭 시 3개 시트로 구성된 엑셀 파일 다운로드
  - 시트1: KPI 통계 (조회기간 + 7개 항목/값)
  - 시트2: 지사별 현황 (14컬럼)
  - 시트3: 상담사별 현황 (17컬럼)
- 2_3) 파일명: `상담통계_조회기간.xlsx`

**3. KPI 카드 (7개, 한 줄 표시):**
- 3_1) 총 보유DB: 인트라넷 전체 준회원 DB 수 (모든 매니저 보유DB 합계)
- 3_2) 회원 컨텍수: 기간 내 실제 통화된 고유 회원 수 (부재중 제외, 중복 제거)
- 3_3) 총 콜수: 기간 내 연결된 통화 건수 (부재중 제외, 중복 허용)
- 3_4) 총 방문예약: 상담결과 '방문상담' 건수
- 3_5) 체결 건수: 상담결과 '가입완료' 또는 '가입중' 건수
- 3_6) 총매출금액: 기간 내 모든 결제금액 합 (업그레이드 포함)
- 3_7) 환산매출: 총매출금액 × 0.91 (VAT 제외)
- 3_8) 라벨 전체 볼드 처리

**4. 지사별 현황 테이블 (14컬럼):**
- 4_1) KPI 카드 하단에 배치
- 4_2) 컬럼: 지사 / 인원수 / 보유DB / 콜수 / 컨텍수 / 포기 / 변경 / 방문예약 / 실방문 / 실방문율(%) / 체결 / 체결율(%) / 총매출 / 환산매출
- 4_3) 하단 합계행 자동 계산
- 4_4) 지사: 본사 / 대전 / 대구 / 부산 등

**5. 상담사별 현황 테이블 (17컬럼, 2행 헤더):**
- 5_1) 지사별 현황 테이블 하단에 배치
- 5_2) 1행 헤더: No / 지사 / 부서 / 상담사 / 보유DB / 콜수 / 회원컨텍수 / DB정리(2열) / 방문상담(3열) / 체결(2열) / 매출(3열)
- 5_3) 2행 헤더: 포기(불가) / 변경 / 총방문예약 / 실제방문 / 실방문율(%) / 체결건수 / 체결율(%) / 신규매출(1가입) / 총매출금액 / 환산매출
- 5_4) 상담사명 클릭 시 상담 상세(consult-detail.html) 새 탭 오픈
- 5_5) 하단 합계행 자동 계산
- 5_6) 페이징 없음 (전체 표시)

**6. 매출 계산 로직:**
- 6_1) 결제유형: 일시불 / 계약금 / 중도금 / 잔금 / 업그레이드
- 6_2) 신규매출(1가입) = 업그레이드 제외한 결제금액 합 (일시불 + 계약금 + 중도금 + 잔금)
- 6_3) 총매출금액 = 모든 결제금액 합 (업그레이드 포함)
- 6_4) 환산매출 = 총매출금액 × 0.91 (VAT 제외)
- 6_5) 분할 납부 시 체결건수(1명=1건)와 매출 시점이 다를 수 있음 (정상)
- 6_6) 현재 Mock: 체결 1건 = 300만원 고정 → 향후 Supabase 결제 테이블 SUM으로 교체

**7. 콜수/컨텍수 계산 로직:**
- 7_1) 콜수: 연결된 통화 건수 (부재중 제외, 같은 회원 중복 카운트)
- 7_2) 컨텍수: 연결된 고유 회원 수 (부재중 제외, 같은 회원 1번만 카운트)
- 7_3) 부재중 상태 체크 건은 콜수·컨텍수 모두 제외

**데이터 수집:**
- `MockAssociates.contactHistory` + `localStorage(purples_call_history_{id})` 기간 필터 적용
- 미팅 데이터: `localStorage(purples_meeting_history_{id})`
- 매출 데이터: 향후 Supabase 결제 테이블 (`SUM(amount) WHERE type ≠ '업그레이드' AND date BETWEEN 기간`)

## 7. 정회원 전환 (MN-202)

**전환 모달 구성:**

| 단계 | 입력 항목 | 시스템 처리 |
|------|----------|-----------|
| 1. 프로그램 선택 | 프로그램(전문직/정규 등), 계약형태(인증/횟수/기간) | 가입비 자동 산출 |

| 3. 전자계약 | 계약서 생성 + 전자서명 요청 | 서명 완료 시 Lock |

**비즈니스 룰:**
1. `가입완료` 상태에서만 전환 버튼 활성화
2. 전환 시 TB_REGULAR에 신규 레코드 생성, TB_ASSOCIATE.assoc_id 연결
3. 계약 시작일 = 전산 생성일 (수정 불가, Lock)
4. 만료일 = 프로그램별 룰 엔진 자동 계산

## 8. 상담통계 지표 정의 (`consult-status.html`)

### 8.1 지표 정의표

| 지표명 | 정의 | 산출 기준 | 비고 |
|--------|------|----------|------|
| **회원컨텍수** | 조회 기간 내 상담사가 준회원에게 실제로 컨텍(전화/문자/카톡 등)한 총 횟수 | contactHistory + callHistory 내 기간 내 건수 합산 | 동일 회원 복수 컨텍 포함 |
| **총 방문예약** | 컨텍 결과가 '방문상담'으로 기록된 건수. 오프라인 방문상담 예약이 잡힌 횟수 | 컨텍 결과 = '방문상담'인 건수 카운트 | 실제 방문 여부와 무관 |
| **실제 방문** | 예약된 방문상담 중 실제로 내방하여 오프라인 상담이 진행된 건수 | meetingHistory 내 기간 내 건수 | No-show 제외 |
| **실 방문율(%)** | 방문 예약 대비 실제 방문한 비율. 예약 대비 실제 운영 효율 파악 | (실제 방문 ÷ 총 방문예약) × 100 | 예약 0건일 경우 0% |
| **체결건수** | 컨텍 결과가 '가입완료' 또는 '가입중'으로 기록된 건수. 계약이 체결된 건수 | 컨텍 결과 = '가입완료' 또는 '가입중'인 건수 | 가입중 포함 (결제 진행 중도 체결로 간주) |
| **체결율(%)** | 실제 방문 대비 계약 체결 비율. 영업 전환율의 핵심 지표 | (체결건수 ÷ 실제 방문) × 100 | 실제 방문 0건일 경우 0% |
| **신규매출(1가입)** | 체결 건수에 따른 신규 가입 매출액. 첫 가입 계약만 포함 | 체결건수 × 건당 계약금액 (기본 300만원) | 업그레이드/재계약 제외 |
| **총 매출금액** | 조회 기간 내 발생한 전체 매출의 합계. 신규 + 업그레이드 + 재계약 등 모든 매출 포함 | 모든 체결 건의 계약금액 합산 | VAT 포함 금액 |
| **환산매출** | 총 매출금액에서 부가세(VAT)를 제외한 실제 수익 매출액 | 총 매출금액 × 0.91 (부가세 10/110 제외) | 성과 평가 기준 금액 |

### 8.2 지표 흐름도

```
컨텍 → 방문예약 → 실제방문 → 체결 → 매출
  ↓         ↓          ↓        ↓       ↓
회원컨텍수  총 방문예약  실제 방문  체결건수  총매출금액
                ↓          ↓              ↓
            실 방문율    체결율        환산매출
```

> 💡 환산매출은 상담사 성과 평가의 기준 금액으로 사용됩니다. 총 매출금액이 아닌 환산매출을 기준으로 성과를 측정합니다.
