## 1. 모듈 개요

| 항목 | 내용 |
|------|------|
| **모듈명** | 준회원 관리 (MN-200 ~ MN-202) |
| **대상 사용자** | 상담 매니저, 팀장, 관리자 |

## 2. 회원 상태 정의 (15개 상태)

### 2.1 상태값 목록

| 단계 | 상태명 | 설명 | 자동화 룰 |
|------|--------|------|----------|
| 연락전 | 컨텍전 | 신규 DB 유입, 매니저 배정 및 첫 연락 전 | - |
| 상담진행 | 부재중(미연락) | 통화 미연결 | 5회 부재 → 결번(불가) |
| 상담진행 | 낮음(컨텍) | 가입 의사 낮음 | - |
| 상담진행 | 보통(컨텍) | 일반 상담 진행 | - |
| 상담진행 | 높음(컨텍) | 가입 의사 높음 | - |
| 상담진행 | 장기상담(컨텍) | 장기적 관리 필요 | 90일×2회 → 재컨텍 |
| 방문상담 | 방문상담 | 오프라인 상담 진행 | - |
| 가입/보류 | 가입보류(컨텍) | 가입 결정 연기 | 30일×1회 → 재컨텍 |
| 가입/보류 | 가입중 | 계약/결제 진행 중 | 14일 경고/30일 파기 |
| 가입/보류 | 가입완료 | 정회원 이관 대기 | - |
| 진행불가 | 결번(불가) | 연락처 오류 | 영구 차단 |
| 진행불가 | 기혼(불가) | 법적 서비스 불가 | 영구 차단 |
| 진행불가 | 기타(불가) | 블랙리스트 등 | 영구 차단 (§7장) |
| DB정리 | 변경 | 타 지사 이관 등 | 팀장 승인 필수 |
| 만료 | 기간만료(재컨텍) | 영업 종료 후 재진입 | 출처태그+쿨타임 |

### 2.2 상태 전이 규칙

- `컨텍전` → 첫 통화 시도 → `부재중` 또는 `낮음/보통/높음`
- `부재중` → 5회 누적 → `결번(불가)` (영구)
- `높음/보통` → 미팅 예약 → `방문상담`
- `방문상담` → 가입 진행 → `가입중` → 전액 결제 → `가입완료` → **정회원 전환**
- `장기상담` → 90일×2회 경과 → `기간만료(재컨텍)` + [태그: 장기상담]
- `가입보류` → 30일×1회 경과 → `기간만료(재컨텍)` + [태그: 가입보류]

## 3. 기능 상세

### 3.1 준회원 목록 (MN-200)

**화면 구성:**
- 검색 필터: 이름, 연락처, 상태, 유입채널, 담당매니저, 지사, 등록일 범위
- 상태별 탭 또는 필터: 전체 / 컨텍전 / 상담중 / 방문상담 / 가입보류 / 진행불가
- 목록 컬럼: 이름, 성별, 나이, 연락처, 상태, 유입채널, 담당매니저, 최근컨텍일, 분배일

**기능 요구사항:**
1. 본인 담당 회원만 조회 (MGR) / 소속팀 전체 (TL) / 전체 (ADMIN)
2. 상태별 색상 구분 (진행불가: 빨강, 가입완료: 초록, 보류: 노랑)
3. 목록에서 바로 상태 변경 불가 — 반드시 상세 페이지에서 변경
4. Excel 다운로드 (TL 이상)

### 3.2 준회원 상세 (MN-201)

**화면 구성 (4개 섹션):**

| 섹션 | 내용 | 수정 권한 |
|------|------|----------|
| 기본정보 | 이름, 연락처, 성별, 생년월일, 지역, 학력, 직업 등 | MGR, TL, ADMIN |
| 상담 이력 | 컨텍 로그 리스트 (일시, 방법, 내용, 결과) | MGR (등록), TL (조회) |
| 미팅 관리 | 미팅 예약/결과 (일시, 장소, 결과) | MGR, TL |
| 담당 변경 이력 | 담당 매니저 변경 이력 (사유 포함) | TL, ADMIN |

**비즈니스 룰:**
1. **상태 변경**: 드롭다운으로 변경 + 사유 입력 필수
2. **부재중 카운트**: 컨텍 결과 "부재" 등록 시 자동 카운트 → 5회 도달 시 `결번(불가)` 자동 전환
3. **장기상담 타이머**: 상태 진입일 기준 90일 카운트, 팝업 경고
4. **담당 변경**: 사유 필수 입력, TL 이상만 실행 가능, 이력 자동 기록
5. **컨텍 등록**: 통화/SMS/카톡/부재 중 선택, 내용 텍스트, 결과상태 선택

### 3.3 정회원 전환 (MN-202)

**전환 모달 구성:**

| 단계 | 입력 항목 | 시스템 처리 |
|------|----------|-----------|
| 1. 프로그램 선택 | 프로그램(전문직/정규 등), 계약형태(인증/횟수/기간) | 가입비 자동 산출 |
| 2. 결제 정보 | 결제수단, 금액, 분할 여부 | 1차 가입비 산정 |
| 3. 전자계약 | 계약서 생성 + 전자서명 요청 | 서명 완료 시 Lock |
| 4. 전산 생성 | 인포팀 확인 | 계약 시작일 = 생성일 Lock |

**비즈니스 룰:**
1. `가입완료` 상태에서만 전환 버튼 활성화
2. 전환 시 TB_REGULAR에 신규 레코드 생성, TB_ASSOCIATE.assoc_id 연결
3. 계약 시작일 = 전산 생성일 (수정 불가, Lock)
4. 만료일 = 프로그램별 룰 엔진 자동 계산
