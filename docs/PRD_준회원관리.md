## 2. 회원 상태 정의 (15개 상태)

### 2.1 상태값 목록

| 단계 | 상태명 | 설명 | 자동화 룰 |
|------|--------|------|----------|
| 연락전 | 컨텍전 | 신규 DB 유입, 매니저 배정 및 첫 연락 전 | - |
| 상담진행 | 부재중(미연락) | 통화 미연결 | 3회 부재 → 결번(불가) |
| 상담진행 | 낮음(컨텍) | 가입 의사 낮음 | - |
| 상담진행 | 보통(컨텍) | 일반 상담 진행 | - |
| 상담진행 | 높음(컨텍) | 가입 의사 높음 | - |
| 상담진행 | 장기상담(컨텍) | 장기적 관리 필요 | 최종컨텍일 3개월 경과 → 컨텍전 원복 |
| 방문상담 | 방문상담 | 오프라인 상담 진행 | - |
| 가입/보류 | 가입보류(컨텍) | 가입 결정 연기 | 최종컨텍일 3개월 경과 → 컨텍전 원복 |
| 가입/보류 | 가입중 | 계약/결제 진행 중 | 14일 경고/30일 파기 |
| 가입/보류 | 가입완료 | 정회원 이관 대기 | - |
| 진행불가 | 결번(불가) | 연락처 오류 | 영구 차단 |
| 진행불가 | 기혼(불가) | 법적 서비스 불가 | 영구 차단 |
| 진행불가 | 기타(불가) | 블랙리스트 등 | 영구 차단 (§7장) |
| DB정리 | 변경 | 타 지사 이관 등 | 팀장 승인 필수 |
| 만료 | 기간만료(재컨텍) | 영업 종료 후 재진입 | 출처태그+쿨타임 |

### 2.2 상태 전이 규칙

- `컨텍전` → 첫 통화 시도 → `부재중` 또는 `낮음/보통/높음`
- `부재중` → 3회 누적 → `결번(불가)` (영구)
- `높음/보통` → 미팅 예약 → `방문상담`
- `방문상담` → 가입 진행 → `가입중` → 전액 결제 → `가입완료` → **정회원 전환**
- `장기상담` / `가입보류` → 최종컨텍일 기준 **3개월 경과** → `컨텍전` 상태로 자동 원복

## 3. 기능 상세

### 3.1 준회원 목록 (list.html)

| 항목 | 내용 |
|------|------|
| **파일** | `pages/associate/list.html` |
| **페이지 ID** | `associate-list` |
| **목적** | 전체 준회원 목록 조회 및 개별 상세 접근 |

**검색 필터:**
- 이름, 연락처, 상태, 유입채널, 담당매니저, 지사, 등록일 범위

**테이블 컬럼:**
| No | 이름 | 성별 | 나이 | 연락처 | 상태 | 유입채널 | 담당매니저 | 최근컨텍일 | 분배일 |

**기능:**
1. 이름 클릭 → 준회원 상세(detail.html) 새 탭 열기
2. 상태별 필터링
3. 페이지당 20건 + 페이지네이션
4. 본인 담당 회원만 조회(MGR) / 소속팀 전체(TL) / 전체(ADMIN)
5. Excel 다운로드 (TL 이상)

### 3.3 상담내역 조회 (consult.html)

| 항목 | 내용 |
|------|------|
| **파일** | `pages/associate/consult.html` |
| **페이지 ID** | `associate-consult` |
| **목적** | 전체 상담 기록 통합 조회 및 신규 등록 |

**검색 필터:**
- 기간 (시작~종료) + 빠른 선택 (오늘/이번주/이번달)
- 지사 드롭다운, 매니저 드롭다운
- 검색 대상 토글 (회원명/전화번호)

**테이블 컬럼:**
| No | 상담일 | 매니저 | 회원명 | 연락처 | 상담결과 | 상담내용 |

**데이터 수집 (collectAllLogs):**
1. Mock `contactHistory` (회원별 내장 이력)
2. `localStorage(purples_call_history_{id})` (전화상담 등록분)
3. `localStorage(purples_meeting_history_{id})` (방문상담 이력)

**상담 등록 기능:**
- `+ 상담 기록 등록` 버튼 → 모달
- 입력 항목: 회원 검색(이름/전화번호), 상담일, 상담유형, 상담결과, 상담내용
- 저장 위치: `localStorage(purples_call_history_{memberId})`
- 등록 후 테이블 자동 새로고침

**비즈니스 룰:**
1. 회원 검색은 담당 매니저가 배정된 회원만 대상
2. 상담결과 값: 부재중, 낮음(컨텍), 보통(컨텍), 높음(컨텍), 장기상담(컨텍), 방문상담, 가입보류(컨텍), 가입중, 가입완료
3. 등록된 기록은 상담통계·매니저 활동관리에 실시간 반영

## 4. 데이터 연동

### 4.1 localStorage 키 체계

| 키 패턴 | 용도 | 사용 페이지 |
|---------|------|------------|
| `purples_call_history_{memberId}` | 전화상담 기록 | detail, consult, consult-status, db-status, consult-detail |
| `purples_meeting_history_{memberId}` | 미팅/방문 기록 | detail, consult, consult-status, visit, consult-detail |
| `purples_status_updates` | 회원 상태 변경분 | 전체 페이지 |
| `purples_visit_data` | 방문 데이터 (가입 관련) | consult-detail |

### 4.2 데이터 흐름

```
준회원 상세 (전화상담 등록 / 방문예약)
        ↓  localStorage 저장
        ↓
상담내역 조회 (상담 기록 등록)
        ↓  localStorage 저장
        ↓
  ┌─────────────────────────┐
  │ purples_call_history_*  │ ← 공통 데이터 소스
  │ purples_meeting_history_*│
  └──────────┬──────────────┘
             ↓ 읽기
  ┌──────────┴──────────────┐
  │ 상담통계                  │ 기간별 매니저 실적 집계
  │ 상담내역 조회              │ 전체 기록 통합 리스트
  │ 매니저 활동관리            │ 매니저별 성과 지표
  │ 매니저 상세                │ 기간별 추이 + 회원목록 + 상담기록
  │ 방문상담 관리              │ 미팅 예약/상태
  └──────────────────────────┘
```

## 7. 정회원 전환 (MN-202)

**전환 모달 구성:**

| 단계 | 입력 항목 | 시스템 처리 |
|------|----------|-----------|
| 1. 프로그램 선택 | 프로그램(전문직/정규 등), 계약형태(인증/횟수/기간) | 가입비 자동 산출 |
| 2. 결제 정보 | 결제수단, 금액, 분할 여부 | 1차 가입비 산정 |
| 3. 전자계약 | 계약서 생성 + 전자서명 요청 | 서명 완료 시 Lock |
| 4. 전산 생성 | 인포팀 확인 | 계약 시작일 = 생성일 Lock |

**비즈니스 룰:**
1. `가입완료` 상태에서만 전환 버튼 활성화
2. 전환 시 TB_REGULAR에 신규 레코드 생성, TB_ASSOCIATE.assoc_id 연결
3. 계약 시작일 = 전산 생성일 (수정 불가, Lock)
4. 만료일 = 프로그램별 룰 엔진 자동 계산
