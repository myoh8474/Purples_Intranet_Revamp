> 버전: 1.0 | 작성일: 2026-06-15 | 근거: 2026.06.11(목) 회원분배회의 (53분 25초)

## 준회원 페이지의 3대 핵심 포인트

### ① DB가 제대로 분배·처리될 것

- 신규 유입 DB가 정확한 지사·정확한 매니저에게 빠짐없이 도달해야 함
- 수동 분배(홈페이지 유입)와 자동 분배(랜딩 페이지 유입)의 이원화 구조 확정
- 분배 순서의 공정성 보장: 매출 순위 기반 순차 배정(라운드 로빈)
- 중복·오류 DB의 자동 감지 및 차단: 같은 경로 + 같은 번호 중복 유입 시 1건만 등록
- 불가/변경 DB의 체계적 후처리 자동화: 결번·기타·기혼은 소스외 처리, 변경은 지사 보관 후 재활용

### ② 매니저가 용이하게 컨택할 수 있을 것

- 매니저 로그인 시 화면 구성: ❶ 방문 상담 예약 → ❷ 전화 상담 예약 → ❸ 내 보유 회원 리스트
- 전화 상담 내역 기록이 가장 중요한 데이터 — 3개월 미컨택 자동 회수의 단일 판단 기준
- 부재중·미컨택 DB를 정렬·관리할 수 있는 기능 필요
- 매니저 연령대(40~50대) 고려한 직관적이고 단순한 UI

### ③ 알림 기능으로 깜빡하지 않게 할 것

- 방문 상담 예약 전일/당일 선알림
- 전화 상담 예약 시간 전 알림
- 부재중 미컨택 DB 리마인더
- 정책 변경 경고 알림 → **제외** (현장 혼란 유발)

## As-Is vs To-Be 개선 리스트

| 구분 | As-Is (현재 시스템 한계) | To-Be (차세대 시스템 개선 요건) |
|------|------------------------|-------------------------------|
| **분배 권한 통제** | 지사 영업팀장이 임의로 실적을 밀어주거나, 관리자 부재 시 분배 업무 마비. 마케팅팀 전체 번호 열람으로 유출 리스크 | 권한 중앙화: 상담팀 분배 권한 영구 차단 + 휴가 시 '권한 대행(On/Off)' 도입. 홍보팀 '전화번호 끝자리 마스킹' 필수 적용 |
| **분배 업무 효율화** | 월요일 아침마다 주말 랜딩 DB 수백 건을 기획팀장이 수작업 분배하여 업무 과부하 | 홈페이지 DB만 수동 분배, 랜딩 DB는 '지역 > 학력 > 매출순위' 룰 엔진 자동 분배 |
| **중복 DB 처리** | 동일 번호 유입 시 데이터 '질'을 따지지 않고 무조건 튕겨내 관리 혼선 | 가치 기반 중복 필터링: 단순 랜딩 중복은 자동 삭제, '가입 폼' 중복은 수동 분배 대기열로 이관 |
| **불가 DB 소스외** | 매니저 불가 상태 표시 → 팀장 수동 확인 후 소스외 변경 | 스케줄러 자동: [불가] 상태 시 Batch 자동 감지 → 소스외 변경 (하루 2회) |
| **변경 DB 회수** | 게시판·구두 요청 → 관리자 수동 이관·회수 | 스케줄러 자동: [변경] 설정 시 소속 지사 '컨택전' 공용 DB로 자동 환수 |
| **3개월 미컨택** | 팀장 수동 확인 후 회수, 보호 DB 구분 없음 | 매월 1일 자동: 전화상담 기준 3개월 미컨택 자동 감지 → 지사 보관 이동 (보호 DB 제외) |
| **만료 정회원 귀속** | 매니저 퇴사 시 DB 귀속 기준 모호 | 매출 1순위 릴레이 배정. 시민권자는 르매리/디노블 유입이라도 퍼플스 ID 자동 귀속 |
| **일정 및 화면 동선** | 전화 예약 시간 놓침 빈번 | UI 강제 렌더링 [1.방문상담 > 2.전화예약 > 3.내 DB]. Push 알림 발송 기능 신설 |

## 1. DB 분배 권한 체계

### 1-1. 분배 실행 권한

| 구분 | 권한 | 비고 |
|------|------|------|
| 회장님 | ✅ 분배 가능 | 최종 권한자 |
| 영업기획팀 팀장 | ✅ 분배 가능 | 평일 실질적 분배 담당 |
| 영업총괄 본부장 | ✅ 분배 가능 | 팀장 부재 시 대행 |
| 상담 매니저 | ❌ 열람 불가 | 분배 페이지 자체 미노출 |

### 1-2. 조회 권한

| 구분 | 분배대상 조회 | 전체내역 조회 | 정보공개 범위 |
|------|:---:|:---:|------|
| 회장님/영업기획팀/영업총괄 | ✅ | ✅ | 전체 공개 |
| 홍보팀 | 조회만 | 조회만 | 전화번호 끝4자리 마스킹 |
| 주말근무자(알바) | 제한적 | ❌ | 전화번호 미노출 |
| 상담 매니저 | ❌ | ❌ | - |

> **홍보팀 특이사항**: CPA/메타 광고 등록 건 정상 등록 여부 확인용. 전화번호 마스킹 필수 (예: 010-****-1234)

## 2. 분배 방식: 자동 vs 수동

### 2-1. 수동 분배 대상 (100% 수동)

**홈페이지 유입 경로** (브랜드를 직접 선택한 고객):
- 결혼 테스트 / 무료 상담 / 가입비 견적 / 자녀 상담
- 사유: 학력·연령·매니저 성향·컨택 이력 등 복합적 판단 필요한 '중요DB'
- 팀장이 학력·연령·재혼여부·직업·유입경로를 종합 판단하여 최적의 매니저에게 배정

### 2-2. 자동 분배 대상

**랜딩 페이지 유입 경로** (CPA, 메타 광고 등):
- 지역별 자동 분배, 남녀 구분 없이 순차 배정
- 데이터 품질이 상대적으로 낮아 기계적 분배 가능

### 2-3. 자동 분배 시 예외 처리

| 상황 | 처리 방식 |
|------|----------|
| 이미 배정된 회원이 재유입 | ❌ 자동분배 안 됨 → 분배 페이지에 잔류 → 수동 확인 |
| 기존 매니저가 1개월 내 컨택 중인 중복 건 | 기존 담당 매니저에게 중복 상태로 재분배 |
| 가입 중(서류 미완비) 상태의 DB | ❌ 절대 다른 곳으로 빠지면 안 됨 → 후임 매니저에게 재배정 |

## 3. 자동분배 순서

### 3-1. 서울 본사

- **기준**: 정(순수) 매출 금액 (환산 매출 ❌)
- **갱신 주기**: 매주 월요일
- **갱신 범위**: 당월 1일 ~ 직전 주 일요일까지 누적 매출
- **순서**: 매출 1위자부터 돌아가며 배정 (A→B→C→D→A...)

> 본사는 매주 월요일마다 순위 변동 가능 — 매출을 많이 하면 DB를 더 받는 인센티브 구조

### 3-2. 지방지사 (경기/부산/대구/대전/광주)

- 전월 매출 순위로 A→B→C→D 순서 확정 (한 달간 동일)
- 매주 월요일마다 순서 재시작 (1위자부터)
- 주중에 DB 부족하면 하위 순위 매니저는 못 받음
- **순위 변동 없음** (본사와 차이점)

---

## 4. 자동 분배 조건 우선순위

### 4-1. 랜딩 페이지 분배 조건 체인

1. **1차 조건**: 지사 선택값 있음 → 선택 지사로 배정
2. **2차 조건**: 지사 선택값 없음 → 지역 정보(필수값)로 해당 지사 배정
3. **3차 조건**: 학력 확인 → 인바운드/아웃바운드 배정

### 4-2. 인바운드/아웃바운드 배정 규칙

| 조건 | 서울본사 & 경기지사 | 부산/대구/대전/광주 |
|------|:---:|:---:|
| 대졸 이상 | → 인바운드(상담1팀) | → 인바운드(상담1팀) |
| 전문대졸 | → 아웃바운드(상담2팀) | → 인바운드(상담1팀) ⚠️ |
| 고졸 이하 | → 아웃바운드(상담2팀) | → 아웃바운드(상담2팀) |

> **인바운드(1팀)**: 경력직 매니저, 고액·고학력 DB 배정
> **아웃바운드(2팀)**: 신입 매니저, 가벼운 DB 배정
> 지방지사는 DB가 열악하여 전문대졸도 인바운드에 '추가 확보' 형태로 배정

## 5. DB 상태 유형 및 처리 (불가 / 변경)

### 5-1. 불가 상태 (DB 영업 가치 없음 → 소스 외 변경)

| 상태 | 의미 | 판단 기준 |
|------|------|----------|
| 불가 결번 | 전화번호 미존재 | 최소 3회 통화 시도 후 "없는 번호" 확인 |
| 불가 기타 | 장애인, 신용불량자 등 | 장애인차별금지법 고려 우회 안내 후 처리 |
| 불가 기혼 | 이미 결혼한 회원 | 카카오톡 프로필/전화 통화 시 확인 |

**처리 프로세스**: 매니저 불가 사유 기록 → 불가 상태 변경 → 스케줄러 자동 감지 → 소스외 자동 변경

### 5-2. 변경 상태 (상담사 변경 요청 — DB 재활용 가능)

> "변경"의 의미: 전화번호 변경이 아님! "나한테서 빼주세요, 다른 매니저한테 변경해 주세요"

**변경 처리 핵심 규칙:**

| 규칙 | 설명 |
|------|------|
| 이관 대상 | 변경 요청한 매니저의 소속 지사로 이관 |
| 상태 복원 | 이관된 DB는 "컨택전" 상태로 초기화 |
| 재배정 시점 | 해당 지사에 신규 매니저가 입사했을 때 |
| 핵심 | 변경 DB는 절대 삭제 금지 — 영업 가치가 남아있는 DB |

### 5-3. 매니저 퇴사 시 DB 일괄 이관

- 버튼 클릭으로 일괄 이관 (기존 시스템 구현됨)
- **예외 보호 DB** (이관 대상에서 제외):
  - 가입 완료 (서류 미완비) — 계약금만 걸어놓고 잔금 추적 관리 필요
  - 가입 중 (인증 미완료) — 인증팀 인증 대기 중
  - 매칭 지정 미완료 — 따끈따끈한 DB
  - → 이 DB들은 반드시 후임 매니저에게 직접 재배정

## 6. 중복 DB 처리 로직

| 상황 | 중복 여부 | 처리 |
|------|:---:|------|
| 같은 경로에서 같은 번호 2회 유입 | ✅ | 1건 삭제 (시스템 오류) |
| 다른 경로에서 같은 번호 유입 | ❌ | 정보가 더 많은 경로의 DB를 매니저에게 배정 |
| 이미 배정된 회원이 다른 매니저 지정 상담 신청 | ❌ | 분배 페이지에 잔류 → 수동 판단 |

> 🔒 **시스템 자동 방지 규칙**: 같은 경로 + 같은 전화번호 + 같은 날 = 1건만 분배 페이지에 등록

> ⚠️ **중복 분배의 위험성**: 같은 DB가 다른 매니저에게 배정되면 매출 분쟁 발생. 중립적 위치의 영업기획팀장만 중복 DB 판단 및 분배 가능.

## 7. 기간만료 DB 재분배

- 기존 상담 매니저 재직 중 → 기존 상담 매니저에게 재배정
- 기존 매니저 퇴사 시 → 해당 지사 매출 순위 순차 분배 (A→B→C→D→A...)
- **온라인 DB 분배와 차이점**: 주간 리셋 없음. 순차적으로 계속 밀리는 구조

> 📌 기간만료 DB에는 아이디 코드(브랜드 코드)가 함께 부여되어야 함.
> 시민권자(해외거주자)는 퍼플스 계약서로만 처리 가능 → 디노블 유입이라도 퍼플스 ID로 처리

## 8. 3개월 미컨택 자동 회수

| 항목 | 내용 |
|------|------|
| 실행 시점 | 매월 1일 |
| 판단 기준 | 전화 상담 내역 기록 (최종 컨택일 기준 3개월) |
| 회수 범위 | 불가/변경 제외, 일반 보유 DB 중 미컨택 건 |
| 예외 | 가입 중(서류 미완비) 상태 DB는 회수 대상에서 제외 |

> 💡 미팅 상담 내역 등록 시에도 전화 상담 내역에 중복 기록됨 → 전화 상담 내역이 단일 판단 기준

## 9. 자동화 가능 영역

| 처리 | 자동화 | 방식 | 스케줄 |
|------|:---:|------|------|
| 불가 → 소스 외 변경 | ✅ | 매니저 상태 변경 시 자동 감지 | 하루 2회 (오전/오후) |
| 변경 → 해당 지사로 이관 | ✅ | 인바운드 기준 자동 이동 | 하루 2회 (오전/오후) |
| 이관 시 컨택전 상태 복원 | ✅ | 이관과 동시에 자동 상태 변경 | 이관 시 즉시 |
| 퇴사 매니저 일괄 이관 | ✅ (기구현) | 버튼 처리 방식 | 수동 트리거 |
| 3개월 미컨택 회수 | ✅ | 전화상담 내역 기준 자동 감지 | 매월 1일 |
