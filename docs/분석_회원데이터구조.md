# 회원 데이터 구조 분석 — 가입횟수별 레코드 중복 문제

> [!CAUTION]
> 인트라넷 재구축 시 반드시 선결되어야 할 핵심 사항입니다.

---

## 1. 현재 문제 — 쉽게 말하면?

### 비유: 은행 통장

현재 구조는 마치 **같은 사람이 은행에 갈 때마다 고객카드를 새로 만드는 것**과 같습니다.

```
김퍼플 씨 (UID: U-001)

📋 1가입 → 고객카드 [SE-2024-001] 발급
   프로그램: 골드, 기간: 2024.01~2025.01
   미팅 5회, 소개 10건, 메모 20건 기록됨

📋 2가입 → 고객카드 [SE-2025-042] 새로 발급
   프로그램: 다이아, 기간: 2025.02~2026.02
   미팅 0회부터 다시 시작, 이전 기록 단절
```

**같은 사람인데** 가입할 때마다 새 카드(= 새 회원정보)가 만들어지고,
이전 기록은 별도 카드에 남아있어서 **한눈에 볼 수 없습니다.**

### 이게 왜 문제인가요?

| 문제 | 구체적 상황 |
|------|------------|
| **데이터가 계속 늘어남** | 회원 1,000명이 평균 2번 가입하면 → 레코드 2,000개. 3번이면 3,000개 |
| **이력을 합쳐서 못 봄** | "김퍼플 씨가 지금까지 미팅 몇 회 했지?" → 1가입, 2가입 각각 따로 확인해야 함 |
| **목록에서 같은 사람이 여러 번 나옴** | 회원 목록에 김퍼플이 2줄 나옴 → 운영자 혼란 |
| **매니저 인수인계 어려움** | 새 매니저가 "이 회원 이전에 어떤 분 만났나요?" → 이전 카드를 따로 찾아야 함 |
| **통계가 부정확** | "활동 회원이 몇 명?" → 실제 인원수와 레코드 수가 다름 |

## 2. 해결 방법 — 어떻게 바꿔야 하나?

### 핵심 아이디어: "사람"과 "계약"을 분리

```
[바뀐 구조]

📁 회원 정보 (사람 단위) — 1명 = 1개만 존재
   ├── 이름: 김퍼플
   ├── 연락처: 010-1234-5678
   ├── 주소: 서울시 강남구
   ├── 상담메모: 통합 관리
   └── 현재 가입: 2가입 (활동 중)

   📎 1가입 계약 (SE-2024-001)
   │   프로그램: 골드
   │   기간: 2024.01 ~ 2025.01
   │   미팅: 5회, 소개: 10건
   │   상태: 약정만료
   │
   📎 2가입 계약 (SE-2025-042)
       프로그램: 다이아
       기간: 2025.02 ~ 2026.02
       미팅: 3회, 소개: 5건
       상태: 활동
```

**한마디로:** 사람 정보는 1개만 유지하고, 계약(가입) 정보만 별도로 쌓는 구조

### 바뀌면 뭐가 좋아지나요?

| 현재 | 개선 후 |
|------|---------|
| 김퍼플이 목록에 2줄 나옴 | **1줄만** 나옴 (현재 가입 정보 표시) |
| 이전 미팅 기록 보려면 다른 레코드 찾아야 함 | **한 화면에서** 1가입/2가입 전체 이력 조회 |
| "총 미팅 몇 회?" 답 못 함 | **자동 합산** 가능 (1가입 5회 + 2가입 3회 = 총 8회) |
| 회원 수 통계가 부풀려짐 | **정확한** 실 인원수 집계 |
| 매니저가 이전 상담 내역 못 봄 | **상담메모는 사람 단위**로 통합되어 전부 보임 |

## 3. 인트라넷 화면은 어떻게 바뀌나?

| 화면 | 현재 | 바뀐 후 |
|------|------|---------|
| 회원 목록 | 같은 사람 여러 줄 | 1명 = 1줄 |
| 회원 상세 | 가입마다 별도 페이지 | **1페이지에 가입이력 탭** 통합 |
| 가입이력 모달 | 제한적 정보 | 전체 계약 타임라인 + 상세 |
| 미팅 목록 | 현재 가입분만 | 전체 통합 or 가입별 필터 선택 |

### 좋은 소식

현재 만들고 있는 인트라넷 프론트엔드가 **이미 이 구조를 예상하고 설계**되어 있습니다.
(가입이력 모달에서 1가입/2가입을 나눠서 보여주는 구조)

→ 백엔드 DB만 정리하면 자연스럽게 연결됩니다.
