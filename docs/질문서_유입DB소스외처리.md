## 1. 현재 파악된 소스외 처리 기준

아래와 같이 정리했습니다. **맞는지 확인**과 **누락 항목** 피드백 부탁드립니다.

| # | 소스외 사유 | 판단 시점 | 자동/수동 | 비고 |
|---|-----------|-----------|:---------:|------|
| 1 | 중복 DB (기존 회원과 동일 번호) | DB 유입 시 | 자동 | 기존 매니저에게 추가 컨텍 알림 |
| 2 | 남성 고졸 이하 | DB 유입 시 (정보 있을 때) | 자동 | 여성은 학력 제한 없음 |
| 3 | 인증 단계 부적격 판단 회원 (재유입) | DB 유입 시 | 자동 | 과거 인증 탈락 이력 대조 |
| 4 | 만 65세 이상 | DB 유입 시 (나이 정보 있을 때) | 자동 | 하한선(19세 미만)도 해당? |
| 5 | 탈회 이력 회원 | DB 유입 시 | 자동? | 전부 소스외? 또는 관리자 판단? |
| 6 | 블랙리스트 재유입 | DB 유입 시 | 자동 | 전화번호/이름 해시 대조 |
| 7 | 스팸/허위 DB | DB 유입 시 | 자동 | 전화번호 형식 오류, 테스트 패턴 |

### Q1. 위 기준이 맞습니까? 빠진 소스외 사유가 있으면 알려주세요.

### Q2. 3번 "인증 단계 부적격 판단 회원"의 범위는?
- 서류 위조 적발 → 소스외 (이해됨)
- 기혼 은닉 적발 → 소스외 (이해됨)
- 서류 미제출/반려로 가입 파기 → 재유입 시에도 소스외?
- 단순 서류 보완 기한 초과 → 재유입 시에도 소스외?

### Q3. 나이 제한 기준 정확히?
- 상한: 만 65세 이상 → 소스외 맞습니까?

## 2. 수동 소스외 처리 관련

자동 분류 외에 **수동으로 소스외 처리가 필요한 경우**가 있을 것 같은데, 아래 상황이 맞는지 확인 부탁드립니다.

### Q4. 현재 수동으로 소스외 처리하는 경우가 있습니까?

제가 추측한 케이스입니다:

| # | 상황 | 추측 이유 |
|---|------|-----------|
| A | 매니저가 통화 중 부적격 발견 (기혼, 고졸 남성 등) | 유입 시 정보가 없어서 자동 체크 못한 경우 |
| B | 자동 분류 오탐지 복구 (정상인데 소스외 처리됨) | 동명이인, 번호 재사용 등 |
| C | 특정 캠페인 DB 일괄 정리 | 품질 낮은 채널 DB 일괄 소스외 |
| D | 팀장/관리자 판단에 의한 처리 | 규칙에 없는 특수 상황 |
| E | 탈회 회원의 재가입 승인 시 소스외 복구 | 소스외 → 컨텍전으로 되돌리기 |

→ **이 중 실제로 발생하는 케이스는?**
→ **이 외에 더 있는 케이스는?**
