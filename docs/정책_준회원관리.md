## 2. 비밀회원(비밀상담) 정책

### 2.1 정의
- **비밀상담**: 부모가 자식 모르게 가입한 건, 또는 본인 외 제3자가 대리 가입한 건 등 당사자에게 노출되어서는 안 되는 상담 건
- 데이터 필드: `isSecret` (Boolean)

### 2.2 운영 규칙

| 항목 | 규칙 |
|------|------|
| **상세 페이지** | 회원명 옆에 `비밀상담` 태그 표시 (주황색 배지) |
| **수정 페이지** | 헤더에 비밀회원 체크박스 (기존값 반영, checked) |
| **등록 페이지** | 헤더에 비밀회원 체크박스 (기본 unchecked) |
| **목록 페이지** | 현재 미표시 (향후 아이콘 또는 필터 추가 검토) |
| **SMS 발송** | 비밀회원은 SMS 발송 시 주의 경고 표시 (향후 구현) |
| **데이터 노출** | 비밀회원 정보는 담당 매니저 + TL 이상만 열람 가능 (향후 RBAC 연동) |

### 2.3 향후 개발 가이드
- 현재: `isSecret` 조건과 무관하게 태그 **고정 표시** (프론트엔드 하드코딩)
- 향후: `m.isSecret === true`일 때만 조건부 표시로 전환
- DB 스키마: `TB_ASSOCIATE.is_secret` (BOOLEAN, DEFAULT false)

## 3. 선호통화시간대 정책

### 3.1 정의
- 회원별로 전화 연결이 용이한 시간대를 사전 기록하여 상담 효율을 높이는 필드
- 데이터 필드: `preferCallTime` (String, 쉼표 구분)

### 3.2 시간대 옵션

| 옵션 | 시간 범위 |
|------|----------|
| 오전(09~12) | 09:00 ~ 12:00 |
| 점심(12~14) | 12:00 ~ 14:00 |
| 오후(14~18) | 14:00 ~ 18:00 |
| 저녁(18~21) | 18:00 ~ 21:00 |
| 야간(21시~) | 21:00 이후 |
| 직접입력 | 체크박스 선택 시 텍스트 입력 활성화 |

### 3.3 운영 규칙

| 페이지 | UI 형태 | 비고 |
|--------|---------|------|
| **상세** | 읽기 전용 텍스트 (주황색 강조) | 핸드폰3 행 옆 배치 |
| **수정** | 체크박스 5개 + 직접입력 (복수 선택) | 기존값 체크 상태 반영 |
| **등록** | 체크박스 5개 + 직접입력 (복수 선택) | 기본 미선택 |
| **목록** | 미표시 | - |

### 3.4 저장 형식
- 체크박스 선택값 + 직접입력값을 쉼표(`, `)로 연결하여 문자열 저장
- 예: `"오전(09~12), 저녁(18~21), 13시30분"`
- DB 스키마: `TB_ASSOCIATE.prefer_call_time` (VARCHAR(200))

## 4. 검색 필터 정책

### 4.1 멀티셀렉트 필터

| 필터 항목 | 옵션 소스 | 복수 선택 |
|----------|----------|----------|
| 성별 | 남, 여 | ✅ |
| 결혼여부 | 초혼, 재혼, 사별 | ✅ |
| 학력 | EDUCATIONS 상수 | ✅ |
| 상태 | STATUSES 상수 (15개) | ✅ |
| 가입경로 | CHANNELS 상수 | ✅ |
| 지사 | BRANCHES 공통 상수 | ✅ |
| 담당매니저 | CONSULTANTS 상수 | ✅ |

### 4.2 필터링 로직
- 선택된 값이 **없으면** 해당 필터는 전체 통과 (AND 조건 미적용)
- 선택된 값이 **있으면** `Array.includes()` 방식으로 매칭
- 모든 필터 조건은 **AND** 연산으로 결합
- **전체 초기화** 버튼: 모든 체크박스 해제 + 키워드 입력 초기화

### 4.3 지사 옵션 공통 관리
- `BRANCHES` 상수로 전체 시스템 공통 관리
- 목록: 서울, 부산, 대구, 인천, 광주, 대전, 울산, 수원, 창원, 제주
- 코드가 아닌 **한글 명칭**으로 표시 및 저장

## 5. 연락처 관리 정책

### 5.1 연락처 구성 (5개)

| 연락처 | 유형 | 부가 기능 |
|--------|------|----------|
| 자택연락처 | 일반전화 | - |
| 직장연락처 | 일반전화 | - |
| 핸드폰1 | 휴대전화 | 관계(본인/모친/부친), 대표번호, 수신거부, 중복확인 |
| 핸드폰2 | 휴대전화 | 관계, 대표번호, 수신거부, 중복확인 |
| 핸드폰3 | 휴대전화 | 관계, 대표번호, 수신거부, 중복확인 |

### 5.2 대표번호 정책
- 핸드폰 1~3 중 **1개만** 대표번호 지정 가능 (라디오 버튼)
- 대표번호 = SMS/알림톡 발송 기본 번호
- 미지정 시 핸드폰1이 기본 대표번호

### 5.3 수신거부 정책
- 각 핸드폰별 개별 수신거부 설정 가능
- 수신거부 설정된 번호로는 마케팅 SMS 발송 차단

## 8. 페이지 간 데이터 정합성

```
목록(list) ──클릭──→ 상세(detail) ──수정버튼──→ 수정(edit)
                                                    ↓ 저장
                                                 상세(detail)

목록(list) ──등록버튼──→ 등록(register)
                              ↓ 저장
                           목록(list)
```

- 등록/수정 시 `localStorage` + `MockAssociates` 배열에 동시 반영
- 상세 페이지 진입 시 최신 데이터로 렌더링
- 향후 Supabase API 연동 시 localStorage 로직을 API 호출로 대체
