# 유입DB 소스외 분류 기준

> 버전: 3.0 | 날짜: 2026-06-12 | 근거: 회원분배회의 (2026.06.11)

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| **목적** | 영업 가치가 없는 DB를 체계적으로 분류·제거하여 매니저 DB 품질을 유지 |
| **적용 범위** | 준회원 전체 DB (매니저 보유 DB + 신규 유입 DB) |
| **핵심 원칙** | DB를 함부로 버릴 수 없는 구조. 소스 외 처리는 유일한 공식 제거 경로 |

---

## 2. 소스 외 처리 유형

### 2-1. 불가 결번

| 항목 | 내용 |
|------|------|
| **의미** | 전화번호가 실제로 존재하지 않음 |
| **확인 절차** | 매니저가 **최소 3회 통화 시도** 후 "없는 번호" 안내 확인 |
| **기록 의무** | 3회 통화 시도 기록을 전화 상담 내역에 남겨야 함 |
| **처리자** | 매니저가 "불가 결번" 상태 표시 → **팀장이 소스 외로 수동 변경** |

### 2-2. 불가 기타

| 항목 | 내용 |
|------|------|
| **의미** | 서비스 이용이 불가능한 개인 사유가 있는 경우 |
| **대표 사례** | • 장애인 (장애인차별금지법에 의해 직접 사유 고지 불가) |
|  | • 신용불량자 (가입비 납부 불가) |
| **실제 대응** | "가입비가 너무 비싸요, 회원님" 등으로 우회 안내 후 불가 기타 처리 |
| **처리자** | 매니저가 기록 + 불가 기타 상태 표시 → **팀장이 소스 외로 수동 변경** |

### 2-3. 불가 기혼

| 항목 | 내용 |
|------|------|
| **의미** | 이미 결혼한 회원 (결혼 주선 서비스 대상이 아님) |
| **확인 방법** | • 카카오톡 프로필 확인 (사진, 가족 관계 등) |
|  | • 전화 통화 시 "저 결혼했어요", "와이프한테 혼나요 전화하지 마세요" 등 확인 |
| **처리자** | 매니저가 기록 + 불가 기혼 상태 표시 → **팀장이 소스 외로 수동 변경** |

## 3. 소스 외 처리 프로세스

```
매니저                              영업기획팀 팀장
  │                                      │
  ├─ ① 불가 사유와 근거 기록              │
  ├─ ② DB 상태를 "불가" 로 변경           │
  │   (결번 / 기타 / 기혼)                │
  │                                      │
  │────── 상태 변경 감지 ──────────────────►│
  │                                      ├─ ③ 기록 확인
  │                                      ├─ ④ 소스 외로 수동 변경
  │                                      │
  │              완료                     │
  │◄──────────────────────────────────────┤
  │                                      │
  ▼                                      ▼
소스 외 DB → 더 이상 어떤 매니저에게도 배정되지 않음 (영구 제외)
```

## 4. 자동 소스외 처리 (신규 유입 시점 즉시 차단)

유입 시점에 아래 조건에 해당하면 **즉시 소스외 처리**되어 신규 분배에서 제외됩니다.

| 분류 | 조건 | 비고 |
|------|------|------|
| **학력/성별** | 남성 고졸 이하 | 여성은 학력 제한 없음 |
| **나이** | 19세 미만 / 65세 이상 | 남녀 동일 |
| **형식 오류** | 전화번호 자릿수 누락, 형식 오류 | 스팸/허위 포함 |
| **현재 진행 중 회원** | 상담 진행 중이거나 정회원인 상태에서 동일 번호 유입 | 중복 DB |

### 중복 DB 처리 방식

| 상황 | 처리 |
|------|------|
| 같은 경로에서 같은 번호 2회 유입 | 1건 삭제 (시스템 오류) |
| 다른 경로에서 같은 번호 유입 | 정보가 더 많은 경로의 DB를 매니저에게 배정 |
| 이미 배정된 회원이 다른 매니저 지정 상담 신청 | 분배 페이지에 잔류 → 수동 판단 |
| 기존 매니저가 1개월 내 컨택 중인 중복 건 | 기존 담당 매니저에게 중복 상태로 재분배 |

> 시스템 자동 방지 규칙: **같은 경로 + 같은 전화번호 + 같은 날 = 1건만 분배 페이지에 등록**

## 5. 과거 불가 DB 재유입 시 처리

과거 이력이 있으나 시간이 흐름에 따라 **고객 상황이 변했을 가능성**(이혼, 취업 등)이 존재하므로, 시스템이 자동 차단하지 않고 **과거 이력 데이터를 매핑하여 매니저에게 정상 분배**합니다.

| 분류 | 조건 | 처리 |
|------|------|------|
| **과거 불가 상태 재유입** | 기혼, 결번, 학생 등으로 종결되었던 회원이 재유입 | 정상 분배 + 이력 태그 노출 |
| **인증 반려자 재유입** | 신원 인증 불가 등의 사유로 정회원 등록 반려 후 재유입 | 정상 분배 + 이력 태그 노출 |

### 이력 태그 노출 방식

매니저가 **통화 전 전략을 수립**할 수 있도록, 화면에 과거 데이터 알림(태그)을 매핑하여 노출합니다.

```
예시:
  ⚠️ "과거 '기혼 불가' 사유로 종결 이력 있음 (2025-03-15)"
  ⚠️ "과거 '인증 반려(서류 미제출)' 이력 있음 (2025-08-20)"
```

## 6. 자동화 옵션

| 처리 | 자동화 | 방식 | 스케줄 |
|------|:---:|------|------|
| 불가(결번/기타/기혼) → 소스 외 변경 | ✅ | 매니저가 상태 변경 시 자동 감지 | 하루 2회 (오전/오후) |
| 3개월 미컨택 자동 회수 | ✅ | 전화상담 내역 기준 자동 감지 | 매월 1일 |

> **3개월 미컨택 회수 기준**: 전화 상담 내역 기록 기준, 최종 컨택일로부터 3개월 경과. 불가/변경 제외, 가입 중(서류 미완비) 상태 DB도 회수 대상에서 제외.
