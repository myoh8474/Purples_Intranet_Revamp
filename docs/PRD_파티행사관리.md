# 파티/행사 관리 설계서

> **작성일**: 2026-05-13  
> **상태**: 초안  
> **관련**: 행사관리, 회원관리, 결제관리

---

## 1. 배경 및 현행 문제점

### 1.1 현행 업무 프로세스

```
[회원] 사용자 사이트에서 파티 신청
   ↓
[상담/매칭 매니저] 신청 확인 → 결제 안내 (이니페이/계좌이체 등)
   ↓
[상담/매칭 매니저] 결제 완료 확인 → 인포상담사에게 수동 전달 (메신저/구두)
   ↓
[인포상담사] 인트라넷에 참석자 수동 등록 + 결제정보 입력
```

### 1.2 문제점

| 항목 | 현황 | 문제점 |
|------|------|--------|
| 신청 접수 | 사용자 사이트에서 개별 신청 | 인트라넷과 연동 없음 |
| 정보 전달 | 매니저 → 인포상담사 수동 전달 | 누락·지연 발생, 전달 이력 없음 |
| 결제 관리 | 인포상담사가 수기 입력 | 결제 상태 실시간 파악 불가 |
| 참석자 현황 | 별도 엑셀 관리 | 성비·수금 현황 즉시 확인 어려움 |
| 행사 이력 | 체계적 관리 부재 | 과거 행사 통계·분석 불가 |

### 1.3 개선 방향

**인트라넷 내 파티/행사 관리 모듈 신설**로 행사 정보 조회, 참석자 등록, 결제 관리를 일원화

## 2. 행사 데이터 구조

### 2.1 행사 유형

| 코드 | 유형 | 설명 |
|------|------|------|
| `REGULAR` | 정기파티 | 월 1~2회 정기 개최 싱글 파티 |
| `PREMIUM` | 프리미엄 | 소수 정예 고급 디너/모임 |
| `SPECIAL` | 특별행사 | 시즌·테마별 이벤트 (와인파티, 비치파티 등) |

### 2.2 행사 상태

| 상태 | 설명 |
|------|------|
| 예정 | 행사 일자 도래 전 (신청 가능) |
| 진행중 | 행사 당일 |
| 완료 | 행사 종료 |
| 취소 | 행사 취소 |

## 3. 데이터 모델

### 3.1 행사 테이블

```
Party {
  id            : 행사 ID (PK)
  name          : 행사명
  type          : 행사 유형 (REGULAR/PREMIUM/SPECIAL)
  date          : 행사 일자
  time          : 행사 시간
  location      : 장소
  fee           : 참가비 (원)
  capacity      : 정원 (명)
  status        : 상태 (예정/진행중/완료/취소)
  description   : 행사 설명
  created_at    : 등록일시
  created_by    : 등록자
}
```

### 3.2 참석자 테이블

```
PartyAttendee {
  id            : 참석자 ID (PK)
  party_id      : 행사 ID (FK → Party)
  member_id     : 회원 ID (FK → Member)
  name          : 회원명
  gender        : 성별 (남/여)
  birth_year    : 출생년도
  phone         : 휴대폰번호
  manager       : 담당 매니저
  pay_method    : 결제방법
  pay_amount    : 결제금액
  pay_status    : 결제상태 (완료/미결제)
  reg_by        : 등록자 (인포상담사)
  reg_date      : 등록일
}
```

### 3.3 결제방법

| 코드 | 결제수단 | 설명 |
|------|----------|------|
| `CARD` | 카드결제 | 신용/체크카드 결제 |
| `INIPAY` | 이니페이 | 이니시스 온라인 결제 |
| `BANK` | 무통장입금 | 계좌이체/무통장 |
| `ONLINE` | 온라인결제 | 기타 온라인 PG |
| `VOUCHER` | 상품권 | 상품권 결제 |
| `FREE` | 무료 | 무료 참석 (VIP/이벤트 당첨 등) |

## 4. 업무 플로우

### 4.1 참석인원 등록 플로우

```
[인포상담사] 인트라넷 접속
   ↓
파티 드롭다운에서 행사 선택
   ↓
행사 정보 확인 (일시, 장소, 참가비, 정원, 현재 참석 현황)
   ↓
[+ 참석인원 등록] 클릭 → 등록 모달 오픈
   ↓
회원 검색 (이름/출생년도/아이디/핸드폰번호)
   ↓
검색 결과에서 회원 선택 → 성별/출생년도/휴대폰 자동 표시
   ↓
결제정보 입력 (결제방법, 결제금액)
   ↓
[등록] 클릭 → 참석자 리스트에 추가
```

### 4.2 결제 확인 플로우

```
[상담/매칭 매니저]
   결제 안내 → 회원 결제 → 결제 확인
   ↓
[인포상담사에게 전달]
   회원명, 결제방법, 결제금액, 결제일
   ↓
[인포상담사]
   인트라넷에서 해당 파티 선택 → 참석인원 등록 시 결제정보 함께 입력
```

## 5. UI 구성

### 5.1 메인 화면

| 영역 | 구성요소 | 설명 |
|------|----------|------|
| 상단 | 파티 선택 드롭다운 | `[유형] 행사명 (일자)` 형식 |
| 파티 정보 카드 | 행사 상세 | 행사명, 유형, 일시, 장소, 참가비, 정원, 상태 |
| 참석 통계 | 요약 바 | 남성/여성 수, 결제완료/미결제 수, 수금합계 |
| 참석자 테이블 | 리스트 | No, 회원명, 회원ID, 성별, 출생년도, 결제방법, 결제금액, 결제상태, 등록일 |

### 5.2 참석인원 등록 모달

| 영역 | 구성요소 | 설명 |
|------|----------|------|
| 회원 검색 | 이름 입력 + 조회 버튼 | 실시간 검색 지원 |
| 검색 결과 | 테이블 | 이름, 출생년도, 아이디, 핸드폰번호 4개 항목 표시 |
| 회원정보 | 읽기전용 표시 | 선택 시 성별/출생년도/휴대폰 자동 표시 |
| 결제정보 | 입력 폼 | 결제방법(셀렉트), 결제금액(숫자 입력) |
| 액션 | 등록/취소 버튼 | 등록 시 참석자 리스트에 즉시 반영 |

### 5.3 참석 통계 표시 항목

| 지표 | 계산 |
|------|------|
| 남성 수 | `gender === '남'` 카운트 |
| 여성 수 | `gender === '여'` 카운트 |
| 결제완료 | `payStatus === '완료'` 카운트 |
| 미결제 | 전체 - 결제완료 |
| 수금합계 | 결제완료 건의 `payAmount` 합계 |

## 6. 회원 검색 조건

### 6.1 검색 필드

| 필드 | 검색 방식 | 예시 |
|------|-----------|------|
| 이름 | 부분 일치 (한글) | `김` → 김수진, 김매니저 |
| 출생년도 | 부분 일치 (숫자) | `1997` → 1997년생 전체 |
| 아이디 | 부분 일치 (영문+숫자) | `P2024` → P2024로 시작하는 회원 |
| 핸드폰번호 | 부분 일치 (숫자+하이픈) | `1234` → 번호에 1234 포함 |

### 6.2 검색 제외 조건

- 해당 파티에 **이미 등록된 회원**은 검색 결과에서 제외

---

## 7. 결제 상태 자동 판별 규칙

| 조건 | 결제상태 |
|------|----------|
| 결제방법 선택 + 결제금액 > 0 | **완료** |
| 결제방법 = `무료` | **완료** (금액 무관) |
| 결제방법 미선택 또는 금액 = 0 | **미결제** |

## 9. 향후 확장 계획

| 항목 | 설명 | 우선순위 |
|------|------|----------|
| 행사 CRUD | 행사 생성/수정/삭제 기능 | 높음 |
| 참석자 수정/삭제 | 등록된 참석자 정보 수정 및 취소 처리 | 높음 |
| 사용자 사이트 연동 | 회원 신청 → 인트라넷 자동 반영 | 중간 |
| 결제 자동 연동 | PG사 결제 완료 시 자동 상태 변경 | 중간 |
| 정원 마감 처리 | 정원 도달 시 신청 자동 차단 | 낮음 |
| 매출 통계 연동 | 파티 수금액을 매출 통계에 반영 | 낮음 |
| 참석 이력 관리 | 회원별 과거 파티 참석 이력 조회 | 낮음 |
| 알림 기능 | 행사 D-3, D-1 자동 알림 | 낮음 |
