# 퍼플스 인트라넷 — 권한 관리 명세서

> 최종 수정: 2025-05-22 | RBAC(Role-Based Access Control) 일괄 적용 전 사전 정의 문서

---

## 1. 역할(Role) 정의

| 코드 | 역할명 | 설명 |
|------|--------|------|
| `MGR` | 상담 매니저 | 준/정회원 상담, 매출 입력, 미팅 관리 |
| `TL` | 팀장 | 소속팀 전체 데이터 조회, 승인, 성과 관리 |
| `FIN` | 재무팀 | 매출 확인/수정, 정산, 환불 처리 |
| `MATCH` | 매칭팀 | 매칭 대상자 검색, 프로필 발송, 약속 배정 |
| `CS` | CS팀 | 클레임 접수/처리, 탈회 진행 |
| `HR` | 인사팀 | 매니저 KPI, 급여 산정, 인사 평가 |
| `EXEC` | 경영진 | 전체 데이터 열람, 경영 통계, 리스크 관제 |
| `ADMIN` | 시스템 관리자 | 권한 설정, 시스템 설정, 전체 관리 |

## 4. 데이터 범위 제한 (RLS)

| 역할 | 조회 범위 |
|------|----------|
| `MGR` | 본인 담당 회원만 |
| `TL` | 소속 팀 전체 |
| `FIN` | 전 브랜드 매출 데이터 |
| `MATCH` | 전체 정회원 (매칭 용도) |
| `CS` | 클레임/탈회 건만 |
| `HR` | 매니저 인사 데이터 |
| `EXEC` | 전체 (브랜드 코드별 필터) |
| `ADMIN` | 전체 (제한 없음) |

---

## 5. 특수 권한 규칙

1. **매출 수정**: `FIN` + `ADMIN`만 가능. 수정 시 변경 이력 자동 기록.
2. **쉐어링 비율 설정**: `TL` + `FIN` + `ADMIN`. 설정 후 실매출 자동 재계산.
3. **정산 승인**: `FIN`이 처리, `EXEC`가 최종 승인.
4. **회원 분배**: `TL` + `ADMIN`만 가능. 분배 로그 기록.
5. **탈회 처리**: `CS` 접수 → `TL` 승인 → `FIN` 환불 처리.
6. **경영/전략 메뉴**: `EXEC` + `ADMIN` 전용. 일반 직원 메뉴에 노출 안 함.
