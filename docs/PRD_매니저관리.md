## 2. 기능 상세

### 2.1 보유DB 현황 (db-status.html?tab=db)

| 항목 | 내용 |
|------|------|
| **파일** | `pages/associate/db-status.html?tab=db` |
| **페이지 ID** | `associate-db-overview` |
| **목적** | 브랜드별 매니저 보유 회원(DB) 수 현황 한눈에 파악 |

**페이지 타이틀:** 보유DB 현황

**테이블 구조:**
- **가로 병합 테이블**: 브랜드 3개(퍼플스, 디노블, 르매리)를 하나의 테이블에 가로로 병합
- 각 브랜드별 colspan 헤더 + 소계

```
┌──── 퍼플스 (9) ────┬──── 디노블 (0) ────┬──── 르매리 (22) ───┐
│ 지사 │ 매니저│보유DB│ 지사 │ 매니저│보유DB│ 지사 │ 매니저│보유DB│
├──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│ 본사 │ 김지은│  4   │      │      │      │ 본사 │ 윤채영│  3   │
│ ...  │      │      │      │      │      │ ...  │      │      │
├──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│      │ 소계 │  9   │      │ 소계 │  0   │      │ 소계 │  22  │
└──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘
```

**인터랙션:**
- 매니저명 클릭 → 매니저 상세(consult-detail.html) 새 탭
- 보유DB 숫자 클릭 → DB 상세(db-detail.html) 새 탭

**데이터 집계:**
- `MockAssociates`에서 `consultant` 필드로 매니저별 보유 회원 수 집계
- `brand` 필드로 브랜드 분류

**비즈니스 룰:**
1. DB 수가 0인 매니저는 해당 브랜드 테이블에서 제외
2. 가장 긴 브랜드 기준으로 행 수 맞춤 (빈 셀은 공백 처리)
3. URL 파라미터 `?tab=db` 사용 시 탭 바 숨김 → 독립 페이지 동작

### 2.2 매니저 활동관리 (db-status.html?tab=perf)

| 항목 | 내용 |
|------|------|
| **파일** | `pages/associate/db-status.html?tab=perf` |
| **페이지 ID** | `associate-db-status` |
| **목적** | 매니저별 상담 활동 지표(컨텍수, 콜수, 미팅, 가입) 관리 |

**페이지 타이틀:** 매니저 활동관리

**검색 필터 (search-table):**

| 필터 | 타입 | 옵션 |
|------|------|------|
| 기간 | date range | 시작~종료 + 오늘/이번주/이번달 빠른 선택 |
| 지사 | select | 전체/본사/대전/대구/부산 |
| 상담사 | text input | 상담사명 검색 (includes 매칭) |

**테이블 컬럼:**
| No | 지사 | 매니저 | 컨텍수 | 콜수 | 방문미팅 | 가입건수 | 가입금액 |

**합계행:**
- 각 수치 컬럼 합계

**데이터 집계 (calcPerfStats):**

| 지표 | 수집 소스 | 집계 방법 |
|------|----------|----------|
| 컨텍수 | contactHistory + purples_call_history_* | 전체 건수 카운트 |
| 콜수 | type='통화'/'전화' + purples_call_history_* | 전화 유형만 카운트 |
| 방문미팅 | purples_meeting_history_* | 예약/취소 제외 |
| 가입건수 | result='가입완료'/'가입중' | 해당 결과 카운트 |
| 가입금액 | 가입건수 × 3,000,000 | 임시 고정 금액 |

**인터랙션:**
- 매니저명 클릭 → 매니저 상세(consult-detail.html) 새 탭
- 지사 드롭다운은 `includes` 매칭 (예: "본사" → "퍼플스본사", "디노블본사" 모두 포함)

**비즈니스 룰:**
1. 보유DB 또는 컨텍수가 1 이상인 매니저만 표시
2. 기본 정렬: 컨텍수 내림차순
3. 값이 0인 셀은 muted 색상, 0 초과 시 primary 색상

### 2.3 매니저 상세 (consult-detail.html)

| 항목 | 내용 |
|------|------|
| **파일** | `pages/associate/consult-detail.html` |
| **페이지 ID** | - (새 탭으로 열림) |
| **목적** | 개별 매니저의 기간별 성과 추이, 관리 회원, 상담 기록 통합 조회 |
| **진입 경로** | 보유DB 현황·매니저 활동관리·상담통계에서 매니저명 클릭 |

**URL 파라미터:** `?manager={매니저명}` (URL 인코딩)

**화면 구성 (4개 섹션):**

#### A. 매니저 기본정보
```
[김지은]  퍼플스본사  보유DB: 4명
```
- 이름(h2), 지사, 보유DB 수

#### B. 기간별 성과 추이

**검색 필터:**
- 기간 (시작~종료) + 오늘/이번주/이번달 빠른 선택

**테이블:**
| 날짜 | 컨텍수 | 콜수 | 방문미팅 | 가입건수 | 가입금액 |

- 선택 기간 내 일별 데이터 표시 (최근순)
- 합계행

**데이터 집계 (calcDailyStats):**
- 날짜 범위 생성 → 일별 초기값 → contactHistory + call_history + meeting_history + visit_data 순회

#### C. 관리 회원 목록

**테이블:**
| No | 회원명 | 성별 | 나이 | 지역 | 연락처 | 회원상태 | 등록일 |

- 해당 매니저가 담당하는 전체 회원 목록
- 회원명 클릭 → 준회원 상세(detail.html) 새 탭
- 이름순 정렬

#### D. 상담 기록

**테이블:**
| No | 상담일 | 회원명 | 연락처 | 상담내용 | 회원상태 |

- contactHistory + purples_call_history_* 통합
- 최신순 정렬, 최대 50건 표시
- 회원명 클릭 → 준회원 상세 새 탭

## 3. 공통 데이터 소스

### 3.1 localStorage 키 체계

| 키 패턴 | 용도 | 읽기 페이지 |
|---------|------|------------|
| `purples_call_history_{memberId}` | 전화상담 기록 | 보유DB, 활동관리, 매니저 상세 |
| `purples_meeting_history_{memberId}` | 미팅/방문 기록 | 활동관리, 매니저 상세 |
| `purples_status_updates` | 회원 상태 변경 | 전체 |
| `purples_visit_data` | 방문 데이터 (가입) | 매니저 상세 |

### 3.2 데이터 흐름 다이어그램

```
[준회원 상세] ─── 상담 등록 ──→ localStorage
[상담내역 조회] ─ 기록 등록 ──→ localStorage
                                  │
                    ┌──────────────┤
                    ↓              ↓
            [보유DB 현황]   [매니저 활동관리]
                    │              │
                    └──────┬───────┘
                           ↓
                    [매니저 상세]
                    (추이+회원+기록)
```

## 4. URL 파라미터 규칙

| 파라미터 | 적용 페이지 | 동작 |
|---------|------------|------|
| `?tab=db` | db-status.html | 보유DB 현황만 표시, 탭 바 숨김 |
| `?tab=perf` | db-status.html | 매니저 활동관리만 표시, 탭 바 숨김 |
| 파라미터 없음 | db-status.html | 탭 바 표시, 기본 보유DB 탭 활성 |
| `?manager={name}` | consult-detail.html | 해당 매니저 상세 표시 |
