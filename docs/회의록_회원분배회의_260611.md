```

  → 신규 매니저 입사 시 재배정
```

## 3. 수동 분배 vs 자동 분배 기준 (21:41 ~ 25:08)

### 수동 분배 (홈페이지 유입 DB)

> **참석자 3 (22:24)**: "홈페이지에서 유입되는 모든 정보는 다 수동으로"

| 유입 경로 | 분배 방식 | 이유 |
|-----------|----------|------|
| 결혼 테스트 | 수동 | 고액 DB, 매니저 역량 고려 필요 |
| 무료 상담 | 수동 | 상담 정보 풍부 |
| 가입비 견적 | 수동 | 가입 의사 높음 |
| 자녀 상담 | 수동 | 특수 케이스 |
| 회원 가입 | 수동 | 정보량 많음 (대학교명, 직업 등) |

### 자동 분배 (랜딩 DB)

| 유입 경로 | 분배 방식 | 조건 |
|-----------|----------|------|
| CPA 광고 | 자동 | 지역 → 지사별 배정 |
| 메타 광고 | 자동 | 지역 → 지사별 배정 |

### 중복 유입 시 처리

> **참석자 3 (37:47)**: "이미 배정돼 있는 DB 경우에는 자동화로 분배 처리는 안 되고 그냥 분배 페이지에 남겨지는 구조로 해서 제가 이거를 확인을 할 수 있는 신규 분배 페이지로 가는 거네요."

- **동일 경로 중복**: 오류로 판단 → 1건만 유지, 나머지 자동 삭제
- **다른 경로 중복**: 정보가 더 풍부한 경로의 DB를 매니저에게 배정
- **이미 배정된 DB 재유입**: 자동분배 안 됨 → 신규 분배 페이지에 남겨서 수동 확인

## 4. 자동 분배 로직 — 본사 vs 지방 (25:08 ~ 39:33)

### 🔴 본사 (서울) — 주간 갱신 방식

```
[매출 순위 기준: 전월 확정 → 매주 월요일 누적 갱신]

6월 1일(월) 시작:
  5월 매출 순위 확정 → A > B > C > D
  분배: A → B → C → D → A → B → ...

6월 8일(월) 갱신:
  6/1~6/7 누적 매출 재계산 → B > A > D > C (순위 변동)
  분배: B → A → D → C → B → A → ...

6월 15일(월) 갱신:
  6/1~6/14 누적 매출 재계산 → 순위 재정렬
  분배: 새 순위대로 라운드로빈...
```

> **참석자 3 (26:43)**: "이거를 수동으로 하나씩 다 분배를 하고 있거든요. 터널 증후군도 생겼고요. 목 디스크 생겼고요."

### 🔵 지방지사 (경기/부산/대구/대전/광주) — 월간 확정 방식

```

DB 유입량 < 매니저 수 → 못 받는 매니저 발생 가능
```

| **본사/경기 예외** | - | 전문대졸까지 배정 가능 |
| **지방지사 예외** | 전문대졸까지 인바운드에 배정 | DB 부족으로 고졸만 |

## 9. 기타 확인 사항

### 퇴사 매니저 DB 일괄 이관
- 현재 기능 구현됨 (신 부장 개발)
- 퇴사 매니저 소속 DB → 해당 지사로 일괄 이관

- **해외 시민권자**: 디노블 유입이어도 무조건 퍼플스 계약서, 퍼플스 아이디 코드
