## 1. 회원 정보 수정 권한 (00:00 ~ 05:30)

### 논의 내용
- **문제**: 키 정보를 회원→상담→매칭이 각각 올려서 실제 키와 10cm 차이 나는 사례 발생
- **결정**: 인증팀에서만 기본정보 수정 가능, 센터장/전략기획 팀장은 인증팀 부재 시 보조 수정 가능

### 수정 권한 구조
| 권한자 | 역할 |
|--------|------|
| **인증팀** | 기본정보 수정 주담당 (요청 접수 → 판단 → 수정) |
| **전략기획 팀장(센터장)** | 인증팀 부재 시 수정 가능, 판단 후 처리 |
| **매칭 센터장** | 권한은 있으나 인증팀 확인 후 수정 권장 |
| **상담/매칭 매니저** | 수정 불가 → 요청만 가능 |

### 핵심 요구사항
> **참석자 1 (03:05~04:57)**: "최근에 수정된 정보에 대해서 뭔가 다른 색깔로 이렇게 테두리가 쳐진다거나... 이 정보가 최근에 한 달 이내에 수정이 됐다라는 게 표시가 될 수 있으면 좋겠다"
- **이유**: 직업(재직) 정보가 전산에서 수정됐지만 프로필에는 반영 안 된 경우, 매니저가 인지하고 프로필도 수정할 수 있도록
- **방식**: 수정된 칸에 한 달간 시각적 표시 (색깔/테두리)

## 2. 회원 상태 관리 — 보류·교제·탈회 (05:30 ~ 25:40)

### 2-1. 임시 교제 / 외부 교제

| 구분 | 설명 |
|------|------|
| **교제 중** | 쌍방 묶임, 프로필 발송 불가 |
| **임시 교제** | 한쪽만 걸 수 있음, 프로필 발송 가능 |
| **외부 교제** | 회원이 외부에서 교제 중, 기간 만료 시 그대로 종료 |

- **문제점**: 매니저가 관리 어려운 회원을 외부 교제로 임의 처리 → 후임 매니저 인수인계 시 실제 외부교제인지 파악 불가
- **사례**: 회원이 "외부 교제 한 적 없다" 클레임 발생

### 2-2. 보류 상태

| 보류 유형 | 기간 제한 | 기간 중단 | 비고 |
|-----------|----------|----------|------|
| **임시 보류** | 없음 (무한정) | 중단 안 됨 | 회원 요청 또는 매니저 판단으로 설정 |
| **약정 보류** | 없음 | 중단 안 됨 | 가입 후 바로 쉬겠다는 경우 |
| **장기 보류** | 3개월 | 중단됨 | 서류 제출 필수, 3개월 후에도 수동 해제 필요 |
| **강제 보류** | - | - | 회장님만 설정 가능 |

### 2-3. 탈회 프로세스

```
탈회 언급 → CS 관리 중(신규 상태값 필요) → 탈회금 확정 → 탈회 진행 → 탈회 완료
```

- **핵심 발언 (참석자 1, 19:43)**: "탈회를 회원이 언급하고 만남이 진행되지 않지만 탈회금은 확정되지 않은 상태의 값이 하나 필요하다"
- **탈회 진행**: 탈회 신청서 접수 + 탈회금 확정 + CS 통보 후에만 상태 변경
- **소보원/소송**: 금액 합의 안 될 경우 소보원·소송 진행, 법무팀이 분산해서 소장 접수

### 2-4. 자동화에 대한 결론

> **참석자 2 (23:30)**: "자동으로 할 수 있는 게 하나도 없네요."
> **참석자 1 (23:47)**: "전산에서 모아서 보기 좋고 알림이 가고... 파악할 수 있게 캐치를 할 수 있도록 이게 더 중요하겠네요."

- ❌ 자동 상태 변경: **위험** (케이스 바이 케이스)
- ❌ 자동 알림톡 발송: **위험** (탈회 언급 회원에게 자동 발송 시 클레임)
- ✅ 매니저 대상 알림: **적합** (보류 기간 체크 알림, 컨택 예정일 알림)
- ✅ 업무 캘린더: **적합** (만료 예정, 보류 기간 경과 등 표시)

## 3. 프로필 발송 규칙 (34:35 ~ 46:15)

### 상태별 프로필 발송 가능 여부

| 상태 | 발송 가능 | 비고 |
|------|----------|------|
| 활동 | ✅ | - |
| 임시 보류 | ✅ | 소개용으로 보낼 수 있음 |
| 약정 보류 | ✅ | 좋은 프로필 보내서 활동 유도 가능 |
| 임시 교제 | ✅ | 한쪽만 걸린 경우 |
| **교제 중** | ❌ | 쌍방 묶여 있으므로 불가 |
| **장기 보류** | ❌ | - |
| **강제 보류** | ❌ | - |
| **소송 중** | ❌ | - |
| **만료 (일자 경과)** | ❌ | **만료를 안 쳐도 만료일자 기준 차단** |

### 만료일 경과 시 처리 (핵심)

> **참석자 1 (37:20)**: "만료일이 지나면 어쨌든 프로필 발송이 안 되게는 돼야 될 것 같아요."
> **참석자 1 (40:54)**: "만료 일자가 지났는데 매니저가 만료를 안 치고 활동 상태로 남아 있는 동안에 만남을 새로 또 잡아주는 경우들이 생기기 때문에"

- 만료를 수동으로 치지 않는 이유: 후기 등록 때문
- **결정**: 만료 상태와 무관하게 **만료일자 경과 시 프로필 발송 + 만남 등록 자동 차단**
- 후기 등록은 만료일 이후에도 가능하도록 별도 처리

### 프로필 응답 처리

- ❌ **자동 거절 처리 반대**: 해외 회원 2개월 뒤 수락 사례 있음
- ✅ **미응답/응답지연 상태** 도입 → 거절이 아닌 보류 형태로 관리
- 장기적으로 매칭 어플 도입 시: 프로필 유효기간 → 만료 → 재발송 요청 버튼
