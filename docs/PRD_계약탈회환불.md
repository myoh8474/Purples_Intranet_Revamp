| 항목 | 내용 |

## 2. 클레임 접수/처리 (MN-700)

### 2.1 접수 프로세스

1. 접수 정보 기록 (일시, 채널, 내용)
2. 유형 자동 분류: 서비스 불만 / 매칭 불만 / 환불 요청 / 기타
3. CS 담당자 자동 배정
4. 1차 상담 진행

### 2.2 에스컬레이션

| 단계 | 처리 |
|------|------|
| 1차 해결 | 해결 조치 기록 → 만족도 조사 → 종결 |
| 미해결 | 팀장/법무 에스컬레이션 |
| 탈회 요청 | 탈회 처리 파이프라인으로 이관 |

## 3. 탈회 처리 (MN-701)

### 3.1 탈회금 산정 기준

| 단계 | 시점 | 환불 비율 |
|------|------|----------|
| ① 프로필 제공 전 | 계약 성립 후 ~ 프로필 제공 전 | 가입비의 **90%** |
| ② 프로필 제공 후 | 프로필 제공 완료 후 | 가입비의 **85%** |
| ③ 만남일자 확정 후 | 만남 일자 확정 이후 | 가입비의 **80%** |
| ④ 1회 만남 이후 | 첫 만남 진행 이후 | 가입비의 **20% + 잔여횟수 환산** |

### 3.2 1가입/2가입 탈회금 정책

- **1가입 탈회**: 1가입 기준 위약금 공제 후 환불액 산정
- **2가입 탈회**: 미개시 → 전액 환불 / 개시 후 → 잔여횟수 환산
- **1장계약(인증제 단일)**: 전체 기준 적용

### 3.3 실매출 마이너스(-) 처리

- **정의**: 전산 오류가 아닌 실제 환불 완료 금액
- **산출**: (원결제 금액) - (위약금 등 공제액)
- 실매출 조회 화면에서 마이너스 = **환불 완료 건**

### 3.4 탈회 프로세스

| 단계 | 내용 | SLA |
|------|------|-----|
| 1 | 탈회 의사 접수 → [탈회진행] 전환 | - |
| 2 | 법무팀 검토 → 위약금 자동 산정 | - |
| 3 | 환불금 계산 + 방어율 적용 | - |
| 4 | 탈회 신청서 전자서명 | - |
| 5 | 합의서 전자서명 | - |
| 6 | 환불 처리 (정산팀) | **7영업일 이내** |
| 7 | [탈회] 상태 전환 + 안심번호 전체 회수 | - |

> ⚠️ 7영업일 초과 미처리 시 법무/정산팀 **SLA 경고 자동 발송**

### 3.5 접수 차단 대상 (탈회 버튼 Disabled)

활동대기, 만료(기간만료), 자동만료, 약정만료, 탈회(완료) 상태

### 3.6 접수 가능 상태

활동, 임시교제, 교제, 외부교제, 약정보류, 임시보류, 장기보류, 강제보류

## 4. 블랙리스트 정책 (§7장)

### 4.1 등록 사유
- 서류 위조 (학력, 재직, 혼인관계, 재산 증명)
- 신원 불량 (기혼, 가입 기준 미달)

### 4.2 시스템 처리
1. [기타(블랙 등)] 상태 전환 → 영구 차단 모드
2. UID 기반 영구 귀속
3. 연락처/주민번호 해시 대조 → 재가입 원천 차단
4. 검색 결과 영구 제외
5. 데이터 수정 권한 전면 차단

### 4.3 가입 단계별 차단
- **준회원 단계**: 번호+주민번호 자동 중복 체크 → 등록 거부
- **인증 단계**: 서류 위조 발견 → 불가 + [기타(블랙)] 부여
- **활동/사후**: 열람/수정 권한 봉쇄 + 법무팀 알림
