## 1. 인증 프로세스 개요

### 1-1. 인증 목적

정회원 가입 시 회원이 제출한 개인정보(학력, 직장, 혼인 여부 등)의 진위를 관공서 발급 서류를 통해 검증하고, 검증 완료된 회원에 한하여 **"활동" 상태를 부여**하여 매칭 서비스를 개시하는 절차입니다.

### 1-2. 인증 주체 및 권한

| 역할 | 담당 업무 | 비고 |
|------|----------|------|
| **인증팀장** | 서류 검증, 인증 상태 관리, 정보 변경 최종 처리 | 유일한 인증 권한자 |
| **인포팀** | 관공서 방문 → 서류 발급/수거, 회원 기본정보 입력 | 서류 수급 담당 |
| **상담 매니저** | 계약 체결, 회원 서류 인포팀 전달, 사진 업로드 | 계약 현장 |

> **결정 사항 (260616)**  
> 기존에 존재했던 **팀장 → 본부장 → 대표이사** 다단계 승인 절차는 **실질적으로 아무도 확인하지 않아** 전량 폐지합니다.  
> 인증팀장 단독 처리로 변경합니다.

## 2. 인증 서류 정책

### 2-1. 필수 서류 (공통 5종)

| # | 서류명 | 발급처 | 확인 항목 | 비고 |
|---|--------|--------|----------|------|
| 1 | 신분증 | 본인 제출 | 본인 확인, 생년월일 | 주민등록증/운전면허증/여권 |
| 2 | 가족관계증명서 | 주민센터/정부24 | 가족 구성, 형제 관계 | 관공서 발급 |
| 3 | 혼인관계증명서 | 주민센터/정부24 | 혼인 이력, 현재 상태 | 관공서 발급 |
| 4 | 학력증명서 | 해당 학교/학력인증기관 | 최종 학력 확인 | 졸업증명서 또는 재학증명서 |
| 5 | 재직증명서 | 소속 회사/건강보험공단 | 현 직장, 재직 여부 | 건강보험 자격득실로 대체 가능 |

### 2-2. 추가 서류 (등급/프로모션별)

| 구분 | 추가 서류 | 설명 |
|------|----------|------|
| **전문직** | 전문 자격증 사본 | 의사·변호사·회계사 등 해당 면허/자격증 |
| **재력 인증** | 자산 증빙 서류 | 부동산 등기부, 금융자산 증명 등 (고정값 아님, 회원 직접 제출) |
| **특수 프로모션** | 프로모션별 지정 서류 | 가입 프로모션 조건에 따라 별도 안내 |

> **핵심**: 추가 서류는 **고정값이 아닙니다**. 회원이 보유한 자산 유형(부동산, 주식, 코인 등)에 따라 증빙 방식이 다르므로, 회원 직접 제출 방식(수동)을 유지합니다.

### 2-3. 사진

| 항목 | 정책 |
|------|------|
| **업로드 주체** | 상담 매니저 (매칭팀/상담팀) |
| **필수 여부** | **필수** — 사진 미등록 시 매칭 불가 |
| **인증팀 역할** | 사진 내용 검증 없음 (첨부 유무만 확인) |
| **업로드 시점** | 계약 체결 후 매니저가 수시 등록 |

### 2-4. 서류 보관 정책

| 항목 | 정책 |
|------|------|
| **보관 기간** | **최소 5년 이상** (법적 의무) |
| **보관 형태** | 본사: 실물 원본 보관 / 지사: PDF 전달 → 출력 후 서류철 보관 |
| **폐기** | 보관 기간 경과 후 법무팀 승인 하에 폐기 |

## 3. 인증 워크플로우

### 3-1. 신규 가입 인증 흐름

```
[매니저] 계약 체결
    ↓
[매니저] 회원 서류를 인포팀에 전달
    ↓
[인포팀] 관공서 방문 → 필수 서류 발급/수거
    ↓                                          
[인포팀] 서류를 인증팀에 전달                    
    ├── 본사: 실물 원본 전달                    
    └── 지사: PDF 업로드 → 인증팀 출력          
    ↓
[인증팀장] 서류 검증 (실물 대조)
    ↓
[인증팀장] 인증 상태별 체크 처리
    ├── 신분증 ✅
    ├── 가족관계증명서 ✅
    ├── 혼인관계증명서 ✅
    ├── 학력증명서 ✅
    └── 재직증명서 ✅
    ↓
[인증팀장] 전체 필수 서류 완료 → "인증완료" 상태 변경
    ↓
[시스템] 회원 상태 "활동" 전환 → 매칭 서비스 개시
```

### 3-2. 본사 vs 지사 서류 수급 차이

| 구분 | 본사 (서울) | 지사 (지방) |
|------|-----------|-----------|
| **서류 수거** | 인증팀이 직접 관공서 방문 (데일리) | 인포팀이 관공서 방문 (계약 발생 시) |
| **방문 주기** | 매일 (계약 건 유무와 관계없이) | 계약 발생 시 (1~3일 간격) |
| **서류 형태** | 실물 원본 | PDF 업로드 → 인증팀에서 출력 보관 |
| **전달 방식** | 인포팀 → 인증팀 직접 전달 | 인포팀 → 플로우(시스템) 통해 업로드 |

### 3-3. 서류 반려 처리

| 항목 | 정책 |
|------|------|
| **관공서 서류 반려** | 거의 발생하지 않음 |
| **주요 반려 사유** | 고객 미첨부 건 (필수 서류 누락) |
| **반려 후 재제출 기한** | **별도 기한 없음** — 단, 필수 서류 미제출 시 "활동" 전환 불가이므로 자연 해소 |
| **극단적 지연** | 1,000건 중 약 1건 수준 (매니저가 서류를 보관하고 미전달하는 경우) |
| **평균 해소 기간** | 약 15일(보름) 이내 |

## 4. 선배정 정책

### 4-1. 선배정이란

계약 체결 후 **인증이 완료되기 전**이라도, 인증팀장이 회원의 메인 등급을 확인하여 **담당 매칭 매니저를 사전 지정**하는 제도입니다.

### 4-2. 선배정 규칙

| 항목 | 정책 |
|------|------|
| **지정 시점** | 계약 체결 직후 (인증 완료 전) |
| **지정 주체** | 인증팀장 |
| **지정 기준** | 회원의 메인 등급에 맞는 매니저 배정 |
| **미팅 대상일 자동 계산** | 서비스 개시일 기준, 1회/2회/3회 미팅 대상일 자동 산출 |

### 4-3. 미팅 대상일 자동 계산

| 미팅 회차 | 기본 기한 | 성격 |
|-----------|----------|------|
| **1회 미팅** | 서비스 개시일 + 10일 이내 | 최우선 매칭 |
| **2회 미팅** | 1회 미팅 이후 + 약 12일 이내 | 후속 매칭 |
| **3회 미팅** | 2회 미팅 이후 + 약 12일 이내 | 약정 완료 목표 |

> **주의**: 미팅 대상일은 **권고 사항**이며 100% 지켜지지 않습니다.  
> 회원 상황에 따라 앞당겨지거나 초과될 수 있으며, 현재 미준수 시 별도 페널티는 없습니다.  
> **향후 개선**: 미팅 대상일 초과 시 매니저/팀장에게 알림 발송 기능 추가 검토 필요 (매칭팀 협의 대상)

## 5. 재인증 / 추가인증 정책

### 5-1. 핵심 원칙: 재인증은 존재하지 않음

| 항목 | 정책 |
|------|------|
| **활동 중 회원** | 인증 중 상태로 **절대 되돌리지 않음** (미팅 차질 방지) |
| **정보 변경 시** | "추가 인증"으로 처리 — 변경 서류 뒷받침 필요, 상태 변경 없음 |
| **재가입 시** | 전체 서류 **신규 재발급** 후 처음부터 인증 진행 |

### 5-2. 추가 인증 (정보 변경) 프로세스

```
[매칭 매니저/상담 매니저] 회원 정보 변경 요청 접수
    ↓
[인증팀장] 변경 근거 서류 확인
    ├── 서류 확인 완료 → 회원정보 직접 수정
    └── 서류 미비 → 추가 제출 요청
    ↓
[시스템] 회원 상태는 "활동" 유지 (변경 없음)
```

> **중요**: 활동 중 회원의 상태를 "인증중"으로 되돌리면 **미팅을 잡을 수 없게 되므로** 절대 상태 변경하지 않습니다.

### 5-3. 정보 변경 주기적 확인 (향후 도입)

| 항목 | 정책 |
|------|------|
| **발송 주기** | 회원 가입일 기준 **6개월마다** |
| **발송 대상** | "활동" 상태인 전체 회원 |
| **발송 내용** | "직장, 연봉 등 변경 사항이 있으시면 담당 매니저에게 연락해주세요" |
| **발송 채널** | SMS (자동) |
| **구현 상태** | 🔴 미구현 (향후 개발 대상) |

> **배경**: 회원이 직장 변경, 연봉 변동 등을 자진 신고하지 않는 경우가 빈번하며, 이로 인해 잘못된 정보로 매칭이 진행되어 **클레임이 발생**하는 사례가 있습니다.  
> 날짜 기준은 가입일 기준으로 설정하여, 가입 1개월 후 발송되는 비효율을 방지합니다.

### 5-4. 혼인관계 주기적 확인 (법무팀 현행)

| 항목 | 정책 |
|------|------|
| **확인 주체** | 법무팀 |
| **확인 방식** | 혼인관계증명서 주기적 재발행 |
| **확인 목적** | 활동 중 회원의 혼인 여부 변동 감지 |
| **현재 상태** | ✅ 운영 중 |

## 6. 폐지 / 제거 대상 기능

> 260616 인증프로세스 회의에서 확인된 **불필요 기능** 목록입니다.

| # | 기존 기능 | 폐지 사유 | 결정 |
|---|----------|----------|------|
| 1 | **다단계 승인** (팀장→본부장→대표이사) | 아무도 확인하지 않음. 인증팀장 단독 처리 | ❌ 제거 |
| 2 | **인증 신청 게시판** | 사용 이력 없음 | ❌ 제거 |
| 3 | **재인증 상태/게시판** | 재인증 개념 자체가 존재하지 않음 | ❌ 제거 |
| 4 | **미인증 확인 게시판** | 인증팀장이 사용하지 않음, 혼란 유발 | ❌ 제거 |
| 5 | **API 자동 인증 (마이데이터)** | 현장 체감 없음 (1건이라도 수동이면 방문 필수), 대리/비밀 가입 예외 존재 | ⏸️ 보류 (1차 제외) |

### 6-1. API 자동 인증 보류 상세 사유

| 보류 사유 | 상세 |
|----------|------|
| **현장 체감 없음** | 10건 중 9건을 자동화해도, 1건만 수동이면 관공서 방문은 여전히 필요 |
| **회원 비협조** | 본인 발급 시도 경험 있으나, 회원들이 잘 안 함 (2024년 하반기 시도 실패) |
| **영업 관행 충돌** | 고액 계약 고객에게 "직접 해드릴게요" 영업 → 자동화와 상충 |
| **대리/비밀 가입** | 본인 인증 불가한 케이스 존재 (부모 대리 가입 등) |
| **결론** | 전체 100% 자동화가 불가한 이상, 현장 체감이 미미하여 1차 개발에서 제외 |

## 7. 인증 상태 관리

### 7-1. 인증 항목별 상태값

| 인증 항목 | 상태값 | 설명 |
|----------|--------|------|
| 신분증 | ⭕ 완료 / ❌ 미완료 | |
| 가족관계증명서 | ⭕ 완료 / ❌ 미완료 | |
| 혼인관계증명서 | ⭕ 완료 / ❌ 미완료 | |
| 학력증명서 | ⭕ 완료 / ❌ 미완료 | |
| 재직증명서 | ⭕ 완료 / ❌ 미완료 | |
| 사진 | ⭕ 첨부됨 / ❌ 미첨부 | 매니저 업로드, 인증팀 관여 없음 |
| 추가 서류 (선택) | ⭕ 완료 / ❌ 미완료 / ➖ 해당없음 | 전문직/재력 등급만 해당 |

### 7-2. 회원 상태 전환 조건

```
[신규] → 계약 체결
    ↓
[인증중] → 필수 서류 5종 + 사진 전체 완료
    ↓
[활동] → 매칭 서비스 개시
```

| 전환 | 조건 | 비고 |
|------|------|------|
| 신규 → 인증중 | 계약 체결 완료 시 자동 | |
| 인증중 → 활동 | 필수 서류 5종 전체 ⭕ + 사진 ⭕ | 인증팀장이 "인증완료" 버튼 클릭 |
| 활동 → 인증중 | **❌ 불가** | 미팅 차질 방지를 위해 절대 역전환 금지 |
